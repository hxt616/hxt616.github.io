4.2

Extends

implements用于子类与接口之间，extends用于基类与子类之间。extends使得子类保留了基类的函数，同时还能在子类中修改继承的函数，还可以添加新的函数。

  • 子类无法访问基类的私有成员
  • 基类的构造函数无法被继承
  • 子类可以通过super.method()直接调用父类中public修饰的函数

在 Java 中，当子类的构造函数被调用时，会先调用父类的构造函数。如果父类没有无参数构造函数，那么必须在子类构造函数中显式调用父类的构造函数，并传递适当的参数。如下所示：

Type Checking（类型检查）

如果基类中的某个方法被子类重写，那么在运行时调用的方法是由该变量的运行时类型（或动态类型）决定的。

  • 静态类型（Static Type）：编译时类型，指编译器在编译代码时知道的类型。它是由变量声明中的类型决定的。

  • 动态类型（Dynamic Type）：运行时类型，指变量在运行时实际引用的对象的类型。
