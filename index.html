LSB隐写原理

发表于 2024-01-30 分类于 图像处理 ， Misc
本文字数： 369 阅读时长 ≈ 1 分钟

本文参考：https://segmentfault.com/a/1190000016223897

先介绍图像相关知识

通道

图像是由很多个像素点组成的，常见的通道数有：单通道、三通道和四通道

  • 单通道：灰度图
  • 三通道：RGB，每个通道代表一个颜色（红、绿、蓝）
  • 四通道：RGBA，Alpha通道表示透明度

位深度

位深度即颜色深度，常用的颜色深度是1位、8位、24位和32位。图像的色彩越丰富，“位”就越多，例如 8 位可表示 256 中颜色深度。

LSB隐写

LSB（最低有效位）全称是Least Significant Bit ，是一种基于图片最低有效位修改储存信息的隐写方法。通过修改RGB颜色分量的最低位二进制（LSB），来进行信息的储存，人类的眼睛并不能感知到前后的变化。

两数之和

本文字数： 879 阅读时长 ≈ 1 分钟

题目地址：https://leetcode.cn/problems/two-sum/

这题用到了哈希表，相关概念参考这篇文章

这题用 C++ 比较方便，需要用 map(key,value)，

  • 数组的大小是受限制的，而且如果元素很少，而哈希值太大会造成内存空间的浪费。
  • set是一个集合，里面放的元素只能是一个key，而两数之和这道题目，不仅要判断y是否存在而且还要记录y的下标位置，因为要返回x 和 y的下标。所以set 也不能用。

步骤：

  • 遍历 nums，寻找target-nums[i]；
  • 若找不到，则将 nums[i] 添加到map中；
  • 否则返回target-nums[i]元素(key)对应的下标(value)；

C++ ：

  • auto 是一个关键字，用于自动推断变量的数据类型。它可以让编译器根据变量的初始化表达式来推断其类型，从而减少代码中的类型声明部分，使代码更加简洁和易读。

  • 关联容器（如std::map）的end()函数返回一个迭代器，指向容器中最后一个元素之后的位置。它并不指向任何有效的元素，本质上是一个“越过末尾”的迭代器。

快乐数

本文字数： 614 阅读时长 ≈ 1 分钟

题目地址：https://leetcode.cn/problems/happy-number/

根据题目描述可知，返回false的条件是该正整数不是快乐数即陷入死循环，但是死循环又无法通过程序返回，因此每一个正整数n都需要一个哈希表，记录每一步的sum，然后判断哈希表中是否重复出现过该值，若出现过则会陷入死循环，则立即返回false；否则若sum==1，则返回true。

n 的取值范围是 1~231-1 ，转换为十进制可知 n 有 10位 数，哈希表(这题是数组)的大小范围为就是 sum 的范围，如下：

1 <= sum <= 1 + 81 * 9 = 730

所以数组定义为 a[731]（下标为0~730）

哈希表

发表于 2024-01-21 分类于 数据结构 ， 哈希表
本文字数： 622 阅读时长 ≈ 1 分钟

  • 散列函数：将关键字映射成该关键字对应的地址的函数（直接映射关系）

  • 散列函数构造方法：

    1. 直接定址法：线性函数 -- H(key) = a × key + b
    2. 除留余数法：最简单最常用--H(key) = key % p（p为最接近/等于散列表长度的质数）
    3. 数字分析法：选取数码分布较为均匀的若干位作为散列地址（例如手机号后四位）
    4. 平方取中法：取关键字的平方值的中间几位数字作为散列地址

  • 处理冲突的方法：

    1. 拉链法（链接法）

    2. 开放定址法

      Hi = ( H(key) + di ) % m

      不能随便物理删除表中元素，若要删除元素，可使用删除标记进行逻辑删除（2023真题）

      • 线性探测法：发生冲突时，顺序查看表中的下一个位置，缺点--大量元素堆积，降低了查找效率
      • 平方探测法：又称二次探测法，散列表长度必须为可以表示为（4j+3）的质数，正负平方交替
      • 伪随机序列法：di 是伪随机数序列
      • 双散列法：使用两个散列函数

  • 装填因子 = 表中记录数/散列表长度

  • 平均查找长度：

    ASL成功 = 查找次数/关键字个数

    ASL失败 = 查找失败次数/p （查找失败次数是从哈希函数计算出的位置[等概率]到第一个空地址(包括)；一共需要计算p个位置的查找失败次数，p为哈希函数所能计算出的地址个数，比如 key%7 的范围是 0~6，所以p为7，不要多算）

    例：https://www.cnblogs.com/47Gamer/p/13160610.html

  • 堆积对存储效率、散列函数和装填因子不会有影响，会影响到平均查找长度

有效的字母异位词

发表于 2024-01-21 分类于 代码随想录 ， LeetCode
本文字数： 656 阅读时长 ≈ 1 分钟

题目地址：https://leetcode.cn/problems/valid-anagram/

这题可以直接用二重循环逐个比较两个字符串中的字符，时间复杂度为 O(n2)，不太好；而使用哈希表的话，时间复杂度为 O(n)，空间复杂度为 O(1)（辅助数组大小为常量26）

思路：

  • 定义一个大小为26的辅助数组，并初始化所有元素为0
  • 'a'~'z' 分别对应辅助数组下标 0~26
  • 遍历 s 字符串，统计每个字符出现次数，即将辅助数组对应下标的元素+1
  • 遍历 t 字符串，将每个字符对应的辅助数组下标的元素-1

s[i]-'a'表示的就是s[i]字符对应的辅助数组下标

环形链表Ⅱ

发表于 2024-01-19 更新于 2024-01-20 分类于 代码随想录 ， LeetCode
本文字数： 914 阅读时长 ≈ 1 分钟

题目地址：https://leetcode.cn/problems/linked-list-cycle-ii/

这题使用快慢指针法，慢指针每次移动一个节点，快指针每次移动两个节点。

分两步解题：

  1. 判断是否有环

    • 若有环，则快慢指针一定相遇，因为快慢指针的相对速度为1，所以快慢指针一定相遇

  2. 找到环的入口

    慢指针走过的节点数为: x + y， 快指针走过的节点数：x + y + n (y + z)，因为快指针是一步走两个节点，慢指针一步走一个节点，所以快指针走过的节点数=慢指针走过的节点数 * 2：

    将 x 移到一边，得到x = n (y + z) - y，下面讨论 n 的值：

    • n = 1 时，x = z

      （1）移动快慢指针，找到相遇点

      （2）定义两个指针，一个指向头节点，另一个指向相遇点；两个指针每次同时移动一个节点直到相遇，该相遇点则为环形入口点

    • n > 1 时，x = (n-1) (y + z) + z

      （1）移动快慢指针，找到相遇点

      （2）定义两个指针 p、q，一个指向头节点，另一个指向相遇点；两个指针每次同时移动一个节点直到相遇，这里与第一种情况不同，就是第二个指针会多转几圈才会与第一个指针相遇，该相遇点则为环形入口点

链表相交

发表于 2024-01-19 分类于 代码随想录 ， LeetCode
本文字数： 925 阅读时长 ≈ 1 分钟

题目地址：https://leetcode.cn/problems/intersection-of-two-linked-lists-lcci/

这题也是408的原题。

解决思路：

  • 分别定义两个指针指向两个字符串首地址
  • 计算出两个字符串的长度
  • 移动长度较大的字符串的指针，使得两个指针指向的节点到尾节点之间的节点个数相同（即链表末尾对齐）
  • 同时移动两个指针，若存在指针相同的节点则返回该节点，否则返回 NULL
