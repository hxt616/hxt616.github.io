KMP算法

原理

  • 时间复杂度：O(n+m)

  • 主串不回退

  • 前缀：除最后一个字符之外的头部子串

  • 后缀：除第一个字符之外的尾部子串

  • 部分匹配值（PM）：最长相等前后缀长度

  • 遇到不匹配的字符时：移动位数 = 已匹配的字符数 - 已匹配的部分匹配值(不匹配前一位的next值)

    模式串指针指向新的位置即pos = next[pos-1]

    next[pos-1]就是已匹配的部分匹配值

  • next 数组：即当前元素对应的该元素前面的子串的部分匹配值（如下图所示）

    若当前元素P[2]与主串不匹配，则模式串的指针跳转到P[next[2]]（即P[0]）

构造next数组：

  • 设模式串为P，假设 next[0], next[1], ... next[x-1] 均已知，求 next[x]

  • 即此时已遍历到模式串 P[x] 元素，需要比较已匹配前缀的后一个字符P[now]与P[x]是否相同：(next[x-1] = now)

    1. 若 P[now] = P[x] ，则 next[x] = next[x-1] + 1

    1. 若 P[now] != P[x] ，则需要缩小 now (即回退已匹配前缀)，now应该改成——使得 A的k-前缀等于B的k-后缀 的最大的k，而B的后缀等于A的后缀（因为A、B是相等的子串），所以只需找到A的k-前缀等于A的k-后缀的最大的k即A的最长公共前后缀的长度，也就是 next[now-1]

    1. 更新 now ：now = next[now-1] ，然后继续比较 P[now] 和 P[x]

IPv6

IPv6特点：

  • 只能在源点分片，传输过程中的路由器不能分片
  • 首部长度不可变

  • 单播：一对一通信，可作为源地址和目的地址

  • 多播：一对多通信，可作为目的地址

  • 任播：一对多中的一个通信，可作为目的地址

IPv4 向 IPv6 过渡：

  • 双协议栈：既支持IPv4，又支持IPv6
  • 隧道技术：使IPv6数据报在IPv4中传输

IPv6可以与DHCP结合使用分配地址，也可以在不使用DHCP服务器的情况下也可以静态分配IPv6：

  • 使用 EUI-64（64位扩展唯一标识）将 MAC地址（48位）转换成64位，再与64位IPv6前缀形成新地址（128位）

  • MAC地址转换成 EUI-64 格式的步骤：

    1. 将二进制形式的MAC地址的第7位（又称U/L bit）进行取反，相当于将BIA[burned-in address]转换成本地管理的地址。

    2. 在MAC地址的前三个字节和后三个字节之间添加fffe

    3. 最后在前面加上IPv6的64位前缀

题目地址：https://leetcode.cn/problems/reverse-words-in-a-string/

这题的难点是去除多余的空格，分为三部分：开头的空格、中间多余的空格、结尾的空格。

  • 先去除开头的空格后，字符串开始位置start就不是0了
  • 去除末尾的空格
  • 在去除中间多余空格时，慢指针要从0开始，将字符串开始位置往前移，同时快指针需要从start开始向后移动直到字符串结尾，这样s就成为了新的没有多余空格的字符串
  • 然后整个字符串翻转
  • 最后每个单词单独翻转，得到结果

先介绍图像相关知识

通道

图像是由很多个像素点组成的，常见的通道数有：单通道、三通道和四通道

  • 单通道：灰度图
  • 三通道：RGB，每个通道代表一个颜色（红、绿、蓝）
  • 四通道：RGBA，Alpha通道表示透明度

位深度

位深度即颜色深度，常用的颜色深度是1位、8位、24位和32位。图像的色彩越丰富，“位”就越多，例如 8 位可表示 256 中颜色深度。

LSB隐写

LSB（最低有效位）全称是Least Significant Bit ，是一种基于图片最低有效位修改储存信息的隐写方法。通过修改RGB颜色分量的最低位二进制（LSB），来进行信息的储存，人类的眼睛并不能感知到前后的变化。

本文字数： 879 阅读时长 ≈ 1 分钟

题目地址：https://leetcode.cn/problems/two-sum/

这题用到了哈希表，相关概念参考这篇文章

这题用 C++ 比较方便，需要用 map(key,value)，

  • 数组的大小是受限制的，而且如果元素很少，而哈希值太大会造成内存空间的浪费。
  • set是一个集合，里面放的元素只能是一个key，而两数之和这道题目，不仅要判断y是否存在而且还要记录y的下标位置，因为要返回x 和 y的下标。所以set 也不能用。

步骤：

  • 遍历 nums，寻找target-nums[i]；
  • 若找不到，则将 nums[i] 添加到map中；
  • 否则返回target-nums[i]元素(key)对应的下标(value)；

C++ ：

  • auto 是一个关键字，用于自动推断变量的数据类型。它可以让编译器根据变量的初始化表达式来推断其类型，从而减少代码中的类型声明部分，使代码更加简洁和易读。

  • 关联容器（如std::map）的end()函数返回一个迭代器，指向容器中最后一个元素之后的位置。它并不指向任何有效的元素，本质上是一个“越过末尾”的迭代器。

快乐数

发表于 2024-01-23 分类于 代码随想录 ， LeetCode
本文字数： 614 阅读时长 ≈ 1 分钟

题目地址：https://leetcode.cn/problems/happy-number/

根据题目描述可知，返回false的条件是该正整数不是快乐数即陷入死循环，但是死循环又无法通过程序返回，因此每一个正整数n都需要一个哈希表，记录每一步的sum，然后判断哈希表中是否重复出现过该值，若出现过则会陷入死循环，则立即返回false；否则若sum==1，则返回true。

n 的取值范围是 1~231-1 ，转换为十进制可知 n 有 10位 数，哈希表(这题是数组)的大小范围为就是 sum 的范围，如下：

1 <= sum <= 1 + 81 * 9 = 730

所以数组定义为 a[731]（下标为0~730）
