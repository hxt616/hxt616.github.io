参考官网：https://picgo.github.io/PicGo-Doc/zh/guide/config.html#github%E5%9B%BE%E5%BA%8A

博客迁移完毕，接下来考虑图床问题。

如果单纯将图片粘贴在Typora中，那个只是保存在本地，部署后，在网页端是无法显示的，这时就需要用到图床，将图片发送到对应的服务器中，然后在文章中以URL的形式展示该图片。

之前我使用的是PicGo，方便好用，准备在新电脑上继续使用，我只需要将之前的配置迁移一下即可。

  • PicGo下载完成后，打开“图床设置”，将对应的仓库名（需要提前创建）和分支填好。
  • 将该仓库的token填写在对应的方框中。
  • 最后一行“设定自定义域名”可以参考这篇文章，简单来说就是Github访问速度慢，需要CDN进行加速。

博客迁移

发表于 2024-06-17 更新于 2024-06-18 分类于 Hexo ， Github
本文字数： 438 阅读时长 ≈ 1 分钟

我的博客方案是：Github Pages + Hexo

参考：https://blog.csdn.net/qq_43698421/article/details/120407042

最近换了新电脑，需要把之前的博客转移到新电脑上。

  1. 首先在新电脑上配置Hexo环境，参考官网，只要按照步骤执行到npm install -g hexo-cli即可（包括这一步）

  2. 将下列文件（位于原电脑中的Hexo文件夹）复制到新电脑中创建好的文件夹中

    • _config.yml
    • package.json
    • scaffolds
    • source
    • themes
    • .gitignore

  3. 执行npm install命令

  4. 执行hexo g，生成静态文件

    这里我有报错，因为新电脑没有Pandoc（文档转换工具），这个工具我主要用于Markdown与LaTeX的转换，方便显示公式。

    解决方案：安装Pandoc以及对应的hexo插件

  5. 执行hexo s，在本地查看是否能正确访问

  6. 若上一步能正确访问，执行hexo d部署网站，然后就可以访问对应的GitHub Pages了

第一步

接着系统会提示你输入文件保存位置。按Enter键接受默认位置（通常是~/.ssh/id_ed25519或~/.ssh/id_rsa）。

第二步

打开GitHub，前往 Settings > SSH and GPG keys ，点击 New SSH key。在 Title 栏输入一个描述性的名称，然后在 Key 栏粘贴之前复制的公钥，最后点击 Add SSH key。

第三步

添加SSH密钥到ssh-agen，当你生成SSH密钥时，通常会设置一个密码短语（passphrase，第一步）来保护你的私钥。每次使用私钥进行身份验证时，例如执行git push、git pull或连接到远程服务器，你都需要输入这个密码短语。通过将SSH密钥添加到ssh-agent，ssh-agent会在内存中缓存解密后的私钥。这样，你只需在将密钥添加到ssh-agent时输入一次密码短语，之后就不需要频繁输入了。

文件共享

硬链接

  • 基于索引结点
  • 不同用户的目录项指向同一个索引结点
  • 链接计数count

软链接

  • 创建Link型(快捷方式)文件——记录文件存放路径
  • 共享文件被删除，Link型文件依然存在，但是会查找失败
  • 软链接索引结点中的引用计数值应该恒为1

建立符号链接时，引用计数不是复制原文件而是必为1

解决方案

vscode插件文件夹位置一般为C:\Users\{username}\.vscode\extensions

    或者将文件夹内的文件剪切/复制到目标文件夹下（即D:\extensions），然后将原文件夹(即C:\Users\{username}\.vscode\extensions)删除

参数 可选/必须 说明
/D 可选 创建目录符号链接而不是文件符号链接（默认为文件符号链接）

今天是3月25日，科软的拟录取名单公布了，终于考上了😃！！！

个人背景

本科就读于浙江的一所双非，专业是计算机科学与技术。大一到大三成绩还行，不过都是靠期末突击，学的也不怎么深。

高数和线代都是大一学的，早忘光了；数据结构期末考了五十几分，还是靠平时分给我拉上来的；计网、操作系统、计组都是大三上学的，学的很浅，期末考试也是靠考前突击过的。

参赛经历：参加过一些大大小小的CTF竞赛，基本只会签到题和一些简单题😓，大三下抱到大腿拿了个国奖。我大学期间就获得过这一个奖项。

择校

一开始考虑的是合工大和杭电，后来无意中加了科软群，本来是看热闹的，后面了解到科软是无导师制的，培养方案很适合我。三月底，看着群里的拟录取截图，备受鼓舞，所以我就下定决心了​​。

初试分数下来一看383，好像还行。后来复试线一出，376分，倒吸一口凉气，我这分数基本就是擦线了，排名也挺靠后，不过好在复试一切顺利。

还是建议大家理性选择，除了考虑地理位置、学校实力、招收人数、培养方案，还要考虑初试复试占比，如果觉得复试没有什么优势，我觉得还是选择6:4和7:3的学校。

经验贴并非现成的复习方案，需要根据自身灵活调整，希望大家理性看待，不要盲目模仿。

下面是我的备考过程

数学二（118分）

推荐老师：高数-武忠祥，线代-李永乐/张宇

推荐教材/习题集：高等数学基础篇、高等数学辅导讲义、线性代数辅导讲义、660、880、高数严选题

推荐模拟卷/真题卷：李林四套卷、李林六套卷、张宇八套卷、张宇四套卷、李艳芳真题

注：我线代基础跟的是张宇，强化跟的是李永乐。推荐大家多听几个老师的课，找到最适合自己。

3月1日~5月20日（基础阶段）

主要任务：（每天三到四小时）

  • 跟着武忠祥学习高等数学基础篇

  • 跟着张宇学习线代基础课（张宇30讲线代部分）

  • 做完对应的书上的例题和习题：高数学习包、张宇30讲线代习题、300题线代部分

  • 高数学完开始学习线代，同时每天抽时间写660高数部分（有难度）

    660我刷了70题就没刷了，感觉有点难，错题太多，有点浪费题目了，所以我在七月份强化阶段的时候才继续从第70题往后刷的

  • 线代学完，可以写写660线代部分（很简单，也可以不写）

  • 有时间可以看看错题，线代例题可以刷两遍

建议：

  • 前期数学可以适当给多一点时间

  • 当老师讲到例题时，可以适当暂停思考一下，再继续。

  • 听完一节课将辅导书上的例题自己单独做一遍

  • 听完一节课就做对应的学习包里的习题

  • 不要赶进度，打牢基础

5月21~10月6日（强化阶段）

强化阶段耗时最长，提升最大

主要任务：（每天四到五小时）

  • 跟着武忠祥学习高数强化课，同时完成严选题
  • 跟着李永乐学习线代强化课，完成书上的习题
  • 在学习高数强化课的时候，最好不要落下线代（可以刷刷之前的错题，或者880中的线代基础题）
  • 完成660高数部分（抽时间写）
  • 完成880：高数+线代（高数部分可以在学习线代强化课的时候开始刷）

我在强化阶段结束时，880的线代部分写到了相似矩阵，所以到十月下旬才完成整本880。根据自己学习进度灵活调整

建议：

  • 听完一节课，将书上的例题再自己写一遍
  • 听完一章的课程，将对应习题集的题目完成：严选题
  • 根据自身学习情况随时调整
  • 有多余时间就二刷错题

10月7日~考前（冲刺阶段）

主要任务：（不超过4.5小时）

  • 7号开始做真题，我做的是09~20年真题，三天两套，第三天订正，这里可以按自己节奏来（我这里留了三年真题到十二月份写）
  • 有时间二刷严选题错题
  • 真题刷完，开始刷模拟卷，我先刷的是23李林四套卷和六套卷（按照考试时间模拟）

  • 23李林模拟卷刷完，开始刷24张宇八套卷
  • 八套卷刷完，刷24李林六套卷和四套卷
  • 最后做张宇四套卷（比较难）

建议：

  • 建议把试卷中的错题记录下来，错误原因、用了什么知识点、什么方法，我是这样记录的：

    • 大题 16：题目要求 y(x) 的区间，而我写成了参数 t 的区间
    • 大题 20：旋转体积算错（倍数不对），第二问物理应用式子没列对

考试当日

我其实没什么感觉，就当平时模拟卷写了，不过前面没进入状态，跳了好几道选填（确实有难度）。大题计算量不小，证明题看完题目就直接跳了，好在线代不难，写完线代就回过头来思考选填，改了两个答案，还有一个填空实在不会。最后写证明题，好像有点思路，硬往泰勒上凑，第一问就胡乱证明了，第二问没思路也没时间了。

primitive type和reference type

发表于 2024-04-05 分类于 Java ， CS61B
本文字数： 1.1k 阅读时长 ≈ 1 分钟

数据类型可以分为两类：基本数据类型（primitive type）和引用数据类型（reference type）。

基本数据类型是指整数、浮点数、布尔值等（byte、short、int、long、 float、 double、 boolean、char）。这些类型的值是直接存储在内存中的，它们的大小是固定的，因此它们的操作速度非常快。

引用数据类型是指对象、数组、函数等类型。这些类型的值是存储在内存中的地址，而不是直接存储在变量中。因此，当我们操作引用类型时，实际上是在操作存储在内存中的地址，而不是实际的值。这使得引用数据类型的操作速度相对较慢。

一个重要的区别是，基本数据类型的值在传递给函数或赋值给另一个变量时，会被复制一份。而引用数据类型的值在传递给函数或赋值给另一个变量时，只会复制它的地址，而不是它的值。这意味着，如果我们修改了一个引用数据类型的值，它会影响到所有引用该值的变量。
