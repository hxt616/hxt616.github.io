六、为什么不直接按列读、按行写？

你可能会想：那我可以改变读取模式来匹配写入吗？

问题：按列读取同样是非合并的，只是把瓶颈从写入转移到了读取。总性能不会改善。

共享内存的优雅之处在于：让读取和写入都变成合并的，通过中间存储解耦了输入和输出的访问模式。

Attention计算和向量变换相关知识点

发表于 2026-01-13 更新于 2026-05-05 分类于 大模型
本文字数： 8.1k 阅读时长 ≈ 7 分钟

向量变换完整流程

1. 基本概念

  • 输入序列 (input_ids): [a, b, c, ...]，长度为 T，batch size 为 B

  • 隐藏状态 (hidden_states): [B, T, H]

  • logits: [B, T, V]，每个位置预测下一个 token

    V 表示词表大小，模型可能的输出类型数，V中的每一个数值（每个维度）对应词表中的一个token，而不是50000维表示一个token

  • KV cache: 每一层的 Key/Value，用于推理阶段避免重复计算

  • Causal mask: 下三角矩阵，保证每个 token 只看当前及之前 token

2. logits 与 token 的对应关系

hidden_state/logits 位置 对应预测 token
logits[:, i, :] input_ids[i+1]
  • 训练阶段: logit[:, i, :] 对应的 label 是下一个 token
  • Prefill 阶段: logit[:, -1, :] 对应最后一个 token 的下一个 token，用于生成
  • Decode / 推理阶段: logit[:, 0, :] 对应当前新 token 的下一个 token

3. 训练 vs 推理阶段 attention 的区别

训练阶段（teacher forcing）

  • 输入完整序列 [a, b, c, d]

  • Q/K/V 全部计算 [B, T, H]

  • attention mask: 下三角矩阵，保证因果性

  • 计算一次 forward 即可，logits [B, T, V]

  • KV cache 不需要，历史 token 重复计算没问题

  • 每个 token 的 Q 与该位置及之前的 K 进行 attention

推理阶段（逐 token decode）

  • Prefill 阶段: 与训练一致，都需要mask，计算完整上下文，生成每层 KV cache

  • Decode 阶段:

    • 输入仅新 token [e]

    • Q 只有新 token

    • K/V = concat(历史 KV + 新 token KV)

    • attention 只计算 新 token的 Q 与所有 K 的 attention

    • causal mask 仍存在，但只有 1 行，实际不起作用

    • logits 只对应 1 个 token

4. KV cache 的作用

  • 训练阶段: 不需要 KV cache
  • 推理阶段:
    • 每一层的 K、V 都需要保存
    • 每层每个时间步的 KV 只服务于同一层的未来 token
    • 上一层 KV 不会覆盖上一时间步的 KV

5. logits shape 总结

阶段 input_ids logits shape 对应预测 token
训练 [a,b,c,d] [B, 4, V] [b,c,d,e]
prefill [a,b,c,d] [B, 4, V] [b,c,d,e]，只取最后一个生成 token
decode [e] [B, 1, V] [f]

MHA/MQA/GQA

发表于 2026-01-07 更新于 2026-05-24 分类于 大模型
本文字数： 2.3k 阅读时长 ≈ 2 分钟

在大型语言模型的推理部署中，KV缓存（Key-Value Cache） 是内存消耗的主要来源。为了优化这一瓶颈，业界经历了从多头注意力（MHA） 到分组查询注意力（GQA） 再到多查询注意力（MQA） 的技术演进。目前 GQA 在主流大模型中更为常用，兼具性能与效率优势。

在标准的Transformer架构中，每个输入token会生成三组向量：

  • Q（Query）：查询向量，决定“关注什么”
  • K（Key）：键向量，提供“被比较的内容”
  • V（Value）：值向量，提供“被提取的信息”

多头注意力（MHA）

MHA的核心特点是：每个注意力头都有自己独立的Q、K、V投影矩阵。这意味着： - 若有32个注意力头，就需要32套不同的K/V投影 - 每个token在每个头上都会产生不同的K/V向量 - 每个头的Q只与该头自己的K/V进行计算

这种设计虽然表达能力最强，但KV缓存开销巨大。对于h个头、d维度的模型，处理长度为L的序列时，KV缓存需要存储2×h×d×L个参数。

多查询注意力（MQA）

MQA是GQA的极端形式：所有Q头共享同一套K/V投影。

关键在于：注意力机制的多样性主要来自Q，而非K/V。

虽然所有Q头看到相同的K和V，但：

  • 每个Q头有不同的查询向量
  • 相同的K与不同的Q计算会产生不同的注意力分数
  • 相同的V用不同的权重组合会产生不同的输出

类比理解：多个分析师使用同一份数据集（K/V），但带着不同的问题（Q）进行分析，最终得到不同的报告。

分组查询注意力（GQA）

研究发现，不同注意力头学习的关注模式存在高度相关性。例如： - 某些头专门关注语法结构 - 某些头专门关注语义相似性
- 某些头专门关注位置关系

这些相关头所需的K/V信息存在大量重叠。GQA基于这一洞察，提出了分组共享的策略。

PPO

近端策略优化，其实就是利用奖励模型的结果更新策略模型参数，既希望语言模型的输出能够尽可能地获得奖励模型的高评价，又不希望语言模型的变化过于“激进”

策略 →（生成）→ 响应 →（输入）→ 奖励模型 →（输出）→ 奖励 →（反馈）→ 策略

概述

在大语言模型（LLM）的强化学习微调（RLHF）中，PPO（近端策略优化）是目前的事实标准。它的核心任务非常明确：利用奖励模型（Reward Model）的反馈，指导策略模型（Actor）生成更符合人类偏好的文本，同时利用价值模型（Critic）来评估生成的质量。

为了让你的读者清晰地理解 PPO，我们将它拆解为以下四个核心步骤。

1. 核心思想：在“探索”与“稳定”之间走钢丝

PPO 的核心逻辑可以用一句话概括：既要让模型根据奖励信号变得更好，又不能让它变得太激进，以免破坏原本学好的语言能力。

为了实现这一点，PPO 引入了裁剪（Clipping）机制。它不会无脑地让模型去追求高分，而是限制模型参数更新的幅度。如果某个生成策略带来的优势（Advantage）过高，PPO 会“剪掉”这部分过高的收益，强迫模型保持克制。

2. 四大核心组件

在 PPO 训练流程中，通常有四个关键的“角色”协同工作：

组件 角色定位 核心职责 是否更新
Actor (策略模型) 演员 负责生成文本（回答）。它是训练的主角，参数会不断优化。 ✅ 是
Critic (价值模型) 预言家 评估 Actor 当前的状态值多少钱（预测未来回报 \(V(s)\)）。 ✅ 是
Reward Model 裁判 根据人类偏好数据训练，给 Actor 生成的完整回答打分。 ❌ 否 (冻结)
Reference Model 守门员 通常是 SFT 后的初始模型，用来计算 KL 散度，防止 Actor 变得太离谱。 ❌ 否 (冻结)

3. 训练流水线：从生成到更新

PPO 的训练过程是一个循环的流水线，我们可以将其分为三个阶段：

阶段一：Rollout (采样与打分) 1. 生成：Actor 模型接收提示词（Prompt），生成一段回答（Response）。 2. 计算 KL 惩罚：计算 Actor 生成的每个 Token 的概率，与 Reference Model 的差异（KL 散度）。这是一个负向奖励，用来约束模型不要偏离常识太远。 3. 最终打分：将生成的完整句子交给 Reward Model，得到一个标量分数（RM Score）。 4. 计算即时奖励：将 RM Score 与 KL 惩罚结合，得到最终的奖励值 \(r_t\)。

阶段二：GAE (计算优势与回报) 这是 PPO 中最容易混淆的部分。我们需要计算两个关键指标： * Advantage (优势) \(A_t\)：表示“这一步做得比平均水平好多少”。 * 我们使用 GAE (广义优势估计) 算法来计算。它结合了蒙特卡洛（MC）的准确性和时序差分（TD）的高效性。 * 公式逻辑：\(A_t = \delta_t + (\gamma \lambda) \delta_{t+1} + \dots\) （其中 \(\delta\) 是 TD Error）。 * Returns (回报) \(G_t\)：表示“从这一步开始，未来能拿到的总分”。 * 它是 Critic 模型训练的“标准答案”。 * 计算公式：\(G_t = A_t + V(s_t)\)。

阶段三：Loss 更新 (策略与价值) 利用计算出的 \(A_t\) 和 \(G_t\)，分别更新 Actor 和 Critic： 1. 更新 Critic：计算预测值 \(V(s_t)\) 与目标值 \(G_t\) 的均方误差（MSE），让“预言家”说得越来越准。 2. 更新 Actor：计算 PPO 的裁剪损失（Clipped Surrogate Loss）。 * 计算新旧策略的概率比 \(r_t(\theta)\)。 * 通过 \(\min(r_t \hat{A}_t, \text{clip}(r_t, 1-\epsilon, 1+\epsilon)\hat{A}_t)\) 来确定最终的梯度方向。

组件

策略模型（Actor）

通常是大型语言模型（如 LLaMA、GPT 等），也就是经过预训练或SFT的LLM，负责生成内容，后续是可以更新参数的

参考模型（Reference）

经过预训练或SFT的模型，与初始的策略模型一致，但是参数冻结，无法更新，目的是为了约束策略模型的更新

奖励模型（Reward）

奖励模型是由人类标注的偏好数据集训练得到的模型，作用是对模型输出进行排序，预测人类偏好，用来代替人类对语言模型的输出进行评价。

提供的是智能体在某一状态下执行某个动作后的即时反馈信号

价值函数（Critic）

用于评估智能体处于某个状态或执行某个 动作-状态对 时的“好坏”，即能够获得的长期回报（累积奖励）

特性 奖励模型 价值函数
是什么 一个函数 R(s, a, s')，表示在状态 s 采取动作 a 到达状态 s’ 后，立即获得的反馈信号。 一个函数 V(s) 或 Q(s, a)，表示从当前状态（或状态-动作对）出发，所能获得的长期、累积回报的期望。
时间视角 短视的、瞬时的。只关心“这一步”的直接结果。 远见的、长期的。关心“从今往后”的所有可能结果。
核心问题 “这个动作现在有多好？” “处于这个状态/采取这个动作，未来能有多好？”
人类类比 感官愉悦/痛苦：吃糖的甜味（+奖励），摸火炉的灼痛（-奖励）。 战略判断：选择读大学（现在辛苦，但长期价值高），选择打游戏（现在快乐，但可能损害长期价值）。
在RL中的角色 环境的反馈机制，定义了任务的终极目标。是价值函数计算的原始输入。 智能体的内部认知模型，是进行决策和规划的核心依据。

更新价值模型参数的过程：

  1. 计算 GAE：算法先通过 GAE 公式，结合奖励和 Critic 的旧预测，算出每一步的 Advantage（给 Actor 用来更新策略）。
  2. 推导 Gt (Returns)：然后，把算出来的 Advantage 加上 Critic 的 旧预测值 (values)，反推出 Returns（即 Gt）。
  3. 计算 Critic Loss：最后，用这个推导出的 Returns ( Gt ) 作为标准答案，去训练 Critic，让它的预测越来越准。critic_loss = (G_t - values)²

returns 的计算方式：

  • Advantage (GAE) = 真实的累计回报 ( Gt ) - 旧的预测值 ( Vt)
  • 真实的累计回报 ( Gt ) = Advantage (GAE) + 旧的预测值 ( Vt)

广义优势估计 (GAE)

它定义了在状态 st（提示 + 到目前为止生成的词）下，特定动作 at（词）相对于策略将采取的平均动作的优越程度

两种计算优势的方法：

  • 蒙特卡洛 (MC)：使用完整轨迹的奖励，高方差低偏差
  • 时序差分 (TD)：每一步都会得到一个即时奖励，低方差高偏差

GAE 的目标就是：在这两者之间取得一个可调节的平衡。通过指数加权平均多个 n-step TD（多步TD） 误差，构造一个偏差与方差可控的优势估计量。

为什么使用GAE而不是用真实的奖励来表示

1. 本质定义的区别

  • Gt（真实的折扣回报）：是“真实的未来”。它代表从当前时刻 t 开始，把未来每一步真实发生的即时奖励（rewards）全部打折并累加起来的总和。
    • 公式： Gt=rt+γrt+1+γ2rt+2+…
  • rewards[t] + gamma * values[t+1]（TD Target）：是“一步真实 + 一步预测”。它只拿了当前这一步的真实奖励 rewards[t]，然后对于未来的部分，它没有去一步步累加真实奖励，而是直接用了 Critic 网络对下一步的预测值 values[t+1] 来代替。

2. 为什么用 TD Target 代替 Gt

在 PPO 或 GAE 的计算中，我们经常看到 rewards[t] + gamma * values[t+1] 这个结构，而不是直接算 Gt*G**t* ，主要有两个原因：

  • 效率更高：计算真实的 Gt 需要等到整个序列结束，把所有未来的奖励都拿齐了才能倒推。而 TD Target 只需要往后看一步就能立刻算出来。
  • 方差更小：真实的 Gt 包含了未来很多步的随机奖励，波动（方差）很大。引入 Critic 的预测值 values[t+1] 相当于加了一个“基准（Baseline）”，能让训练信号更稳定。

3. 它们是如何产生联系的？（GAE 的核心逻辑）

在你之前看到的 GAE（广义优势估计）代码中，它们的关系体现得淋漓尽致： GAE 的核心就是计算每一步的 TD 误差（Delta）： delta_t = rewards[t] + gamma * values[t+1] - values[t]

这里的 rewards[t] + gamma * values[t+1] 就是用来逼近真实回报 Gt*G**t* 的。

  • 如果 Critic 预测得极其完美，那么 values[t+1] 就会非常接近未来真实的折扣回报 Gt+1 ，此时 TD Target 也就几乎等于 Gt 。
  • 正因为 Critic 预测得不完美，所以 GAE 会通过参数 lambda 把多步的 delta 累加起来，以此来不断修正和逼近真实的 Gt 。

总结一下： rewards[t] + gamma * values[t+1] 不等于 Gt​ ，它只是 Gt​ 的一个单步估算版本。真实的 Gt 需要累加未来所有的真实奖励，而这个公式是用“Critic 的预测”偷了个懒，代替了未来繁琐的累加过程。

计算方法

这一步的优势 = 这一步的即时奖励 + 新预测的最终成绩 - 旧预测的最终成绩

  • R_t：在状态 s_t 采取行动后立即获得的奖励
  • V(s_t)：在状态 s_t 预测从此刻到结束能获得的总奖励
  • γ：折现因子（0≤γ≤1），通常接近1
  • V(s_{t+1})：进入新状态后的总奖励预测
λ 取值 含义 等价于
λ = 0 只看当前一步 TD error TD(0)
0 < λ < 1 TD 与 MC 的折中 GAE
λ = 1 看完整轨迹 MC Advantage

A > 0： 👉 这次回答 比 critic 预期的好 👉 应该鼓励 actor 更常这么写

A < 0： 👉 这次回答 比预期的差 👉 应该抑制 actor 再这么写

A ≈ 0： 👉 和预期差不多 👉 别怎么动参数

示例

场景：AI生成句子“我喜欢吃苹果。”

  • 第1步：生成“我”

    • 价值函数（预言家）预测最终奖励：预测分数 = 0.5

  • 第2步：生成“喜欢”

    • 即时奖励（TD）：+0.3（因为“我喜欢”是常见搭配）
    • 价值函数重新预测：预测分数 = 0.8（因为开头不错，提高了最终预期）
    • GAE计算的这一步的优势：0.3（即时奖励）+ 0.8（新预测）- 0.5（旧预测）= +0.6
    • 解读：生成“喜欢”这个词很棒！不仅本身得分高（+0.3），还让预言家对最终结果的预测提升了0.3分，总优势为+0.6。
    • rewards[t]（即时奖励）：对应 +0.3 这是环境（或奖励模型）在你生成“喜欢”这个词后，立刻给出的直接反馈分数。
    • values[t+1]（新预测/下一状态价值）：对应 0.8 这是生成完“喜欢”后，预言家（Critic）站在新的节点上，对未来能拿到的总分的重新预估。
    • values[t]（旧预测/当前状态价值）：对应 0.5 这是在还没生成“喜欢”之前，预言家对这一步的原始预估。

  • 第3步：生成“吃”

    • 即时奖励：+0.2
    • 价值函数重新预测：预测分数 = 0.9
    • 优势：0.2 + 0.9 - 0.8 = +0.3

  • 第4步：生成“苹果”

    • 即时奖励：+0.1
    • 价值函数重新预测：预测分数 = 1.0（句子完整且合理）
    • 优势：0.1 + 1.0 - 0.9 = +0.2

DAPO

在GRPO的基础上进行了调整

https://zhuanlan.zhihu.com/p/31157035727

GRPO 的损失计算包含两个层面的“token 级”操作：

  • 重要性采样权重在 Token 级别：GRPO 的核心损失函数使用重要性采样比率 r(θ)，这个比率是在每个 token 上计算的。公式为： r_{i,t}(θ) = π_θ(o_{i,t} | ...) / π_{θ_old}(o_{i,t} | ...) 这意味着模型在优化时，会对生成序列中的每一个 token，都计算新旧策略下出现该 token 的概率比值。
  • 损失聚合在序列级别：虽然权重在 token 级计算，但最终计算损失时，GRPO 采用先序列内平均，再跨序列平均的方式。其公式为： L = - (1/G) * Σ_i [ (1/|o_i|) * Σ_t loss_{i,t} ] 这种“序列级归一化”意味着每个序列对总损失的贡献权重相同，但序列内部的每个 token 对所属序列损失的贡献也相同。

而DAPO考虑了token对损失的影响，从而有效处理长序列中token的影响

一、大模型量化的分类

1. 按量化粒度分类

（1）权重量化（Weight Quantization）

  • 静态权重量化：推理时权重固定为量化值
  • 动态权重量化：根据输入动态调整权重量化参数

（2）激活量化（Activation Quantization）

  • 静态激活量化：使用校准集预先确定激活值量化参数
  • 动态激活量化：推理时实时计算激活值量化参数

（3）权重-激活协同量化

  • 同时量化权重和激活值，实现端到端的低精度推理

2. 按量化位置分类

（1）后训练量化（Post-Training Quantization, PTQ）

原理：在模型训练完成后，通过统计分析确定量化参数

（2）量化感知训练（Quantization-Aware Training, QAT）

原理：在训练过程中模拟量化效应，让模型适应低精度表示

（3）训练时量化（Quantization During Training）

从训练初期就引入量化约束，直接学习低精度表示

二、最常见的量化方案：MinMax量化算法

对称量化 vs 非对称量化

计算公式：

  • S：缩放因子
  • Z：零值
  • R：待量化的浮点数
  • Q：量化后的值

对称量化

  • 只计算 scale（S）
  • Z = 0

优势： - 实现简单，计算高效 - 零点是固定的0，减少计算开销 - 广泛硬件支持（如NVIDIA Tensor Core）

非对称量化

  • 计算 scale（S）和 zero-point（Z）

优势： - 更好地利用数值范围 - 适合非对称分布的数据

三、大模型量化的关键技术方法

1. GPTQ（GPT Quantization）

原理：基于Hessian矩阵的逐层最优量化

特点： - 针对Transformer架构优化 - 保持注意力机制的关键特性 - 在LLaMA、GPT等模型上效果显著

2. AWQ（Activation-aware Weight Quantization）

原理：激活感知的权重量化

创新点： - 不依赖反向传播或梯度 - 仅需少量校准数据 - 保持原始模型的准确性

3. SmoothQuant

原理：解决激活值异常值问题

效果： - 实现INT8权重量化和INT8激活量化 - 在LLaMA、OPT等模型上实现接近无损量化

4. LLM.int8()

原理：混合精度分解

特点： - 首次在175B参数模型上实现有效8位量化 - 完全保留FP16精度

5. QuaRot（Quantized Rotary Embeddings）

原理：专门针对RoPE位置的量化

优势： - 针对Transformer位置编码优化 - 保持注意力分数的相对顺序

6. ZeroQuant系列

原理：层级联量化与硬件感知优化

题目地址：https://leetcode.cn/problems/car-pooling/description/

问题描述

你是一名出租车司机，车上最初有 capacity 个空座位。车辆只能向一个方向行驶（不允许掉头或改变方向）。给定整数 capacity 和一个行程数组 trips，其中 trips[i] = [numPassengers, from, to] 表示第 i 次行程有 numPassengers 名乘客，上车和下车的位置分别是 from 和 to（这些位置是从汽车的初始位置向东的公里数）。只有当你可以完成所有给定的行程时（即不会出现超载的情况），返回 true，否则返回 false。

解题思路

主要就是判断重叠区间需要搭载的人数是否超载

排序 + 优先队列（最小堆）

这个问题的关键在于我们需要实时跟踪车辆在每个位置上的乘客数量，确保在任何时候乘客数量都不超过车辆的容量。

算法步骤：

  1. 按上车地点排序：首先将所有行程按照乘客的上车地点（from）进行升序排序。这样我们可以按照车辆行驶的顺序处理每个行程。
  2. 使用优先队列跟踪下车事件：我们使用一个最小堆（优先队列）来记录当前车上的乘客组，堆顶元素是最近要下车的乘客组（即下车地点最小的）。堆中存储的是(乘客数量, 下车地点)这样的元素对。
  3. 模拟乘车过程：
    • 遍历排序后的行程，对于每个行程：
      • 检查当前是否有乘客需要下车：如果堆顶元素的下车地点小于等于当前行程的上车地点，说明这些乘客应该在下个行程开始前下车，我们将它们从堆中弹出，并减少当前乘客数量。
      • 将当前行程的乘客加入堆中，并增加当前乘客数量。
      • 检查当前乘客数量是否超过车辆容量，如果超过则返回false。
    • 如果所有行程都能顺利完成，返回true。

  • 卷积层由多个卷积核组成。

例如：将3x3填充到5x5，那么经过2x2卷积核的计算后得到的输出是4x4

步幅

我们将卷积核每次滑动元素的数量称为步幅（stride）。卷积窗口（卷积核）从输入张量的左上角开始，向下、向右滑动。 在前面的例子中，我们默认每次滑动一个元素。 但是，有时候为了高效计算或是缩减采样次数，卷积窗口可以跳过中间位置，每次滑动多个元素。

输入通道和输出通道

  • 输入通道 = 图像的颜色通道
  • 一个卷积核就对应一个输出通道，卷积时，先对每个输入通道做卷积，再把结果相加，得到单个输出通道。
  • 如果输入有 ci 个通道，我们要得到 co 个输出通道，那就需要准备 co 套这样的卷积核。所以总的卷积核形状就是四维的：(co, ci, kh, kw)

1. 输入含义

  • X：输入，多通道特征图（shape = (ci, H, W)）
  • K：卷积核张量（shape = (co, ci, kh, kw)）

其中：

  • ci = 输入通道数
  • co = 输出通道数
  • kh, kw = 卷积核的高和宽

2. 代码逐步解析

（1）列表推导式

  • for k in K
    • 遍历卷积核的第 0 个维度（即输出通道维度 co）
    • 每次 k 的形状是 (ci, kh, kw)
    • 意思是“第 o 个输出通道的卷积核”，它需要和所有输入通道 X 做卷积

（2）调用 corr2d_multi_in

  • 输入：X 的形状 (ci, H, W)
  • 卷积核：k 的形状 (ci, kh, kw)
  • 内部逻辑：
    • 遍历输入的每个通道 x 和对应的卷积核 k[i]
    • 做 2D 卷积（互相关），再相加
  • 输出：单个通道的结果，形状 (H_out, W_out)

所以 corr2d_multi_in(X, k) 的结果就是 一个输出通道。

（3）torch.stack([...], 0)

  • 把所有输出通道的结果沿着新的维度 堆叠起来
  • 输出形状：(co, H_out, W_out)

3. 举个例子

假设：

  • 输入 X：形状 (3, 6, 6)，RGB 三通道
  • 卷积核 K：形状 (2, 3, 3, 3)（2 个输出通道，每个核对应 3 个输入通道）

流程：

  1. 遍历 K[0]（形状 (3, 3, 3)），对 X 做多通道卷积 → 得到输出通道 0
  2. 遍历 K[1]（形状 (3, 3, 3)），对 X 做多通道卷积 → 得到输出通道 1
  3. torch.stack([...], 0) → 最终输出 (2, H_out, W_out)

总结

corr2d_multi_in_out 的功能就是实现 多输入通道 → 多输出通道 的卷积过程：

  • 遍历输出通道 co
  • 每个输出通道对应一组 (ci, kh, kw) 的卷积核
  • 对输入 X 的 ci 个通道做卷积并相加
  • 把所有输出通道结果堆叠起来

最终得到形状 (co, H_out, W_out) 的输出特征图。

1 × 1 卷积层

可以将卷积层看作在每个像素位置应用的全连接层

池化层（pooling layer）

类似于卷积层，池化层也是由一个个形状固定的窗口组成（矩阵），根据步幅滑动，只是计算过程不同

1. 降采样（下采样）

  • 减小特征图的 空间尺寸 (H, W)，比如从 32×32 变成 16×16。
  • 这样可以：
    • 降低计算量（减少后续卷积层的输入规模）。
    • 减少参数量，提高效率。

2. 增强平移不变性

  • 池化在一个小区域里取“代表性”数值（最大值或平均值）。
  • 即使输入图像稍微平移/变形，池化后的特征图变化不会太大。
  • 这使得模型对小范围的 位置变化不敏感。

3. 计算方法

  • 最大池化（Max Pooling）：保留局部区域最强的激活（比如边缘、角点）。
  • 平均池化（Average Pooling）：取区域的平均值，平滑特征图。
  • 本质上就是一种 特征选择 或 特征压缩。

自注意力机制

常见的大语言模型一般只包含解码器，没有编码器

自注意力模型会“吃”整个序列的数据，输入几个向量，它就输出几个向量

对于输入序列中的每个 token（向量），我们会计算出三个矩阵：（X为输入矩阵即多个向量的组合，乘以对应的矩阵得到 Q、K、V）

  • Query (Q)：查询向量，\(Q = W^QX\)
  • Key (K)：键向量，\(K = W^KX\)
  • Value (V)：值向量，\(V = W^VX\)

计算注意力权重（softmax 后）： \[ \text{Attention}(Q, K, V) = \text{softmax}\left(\frac{QK^T}{\sqrt{d_k}}\right)V \] \(d_k\) 是 Key 向量的维度（用于缩放避免梯度过大）

多头注意力机制：

Transformer 实际使用的是 多头自注意力，先计算整个向量的 Q、K、V，再进行拆分，下面是完整步骤

解码器

自回归解码器（最常用）

在生成序列时，每一步仅基于当前及之前的token预测下一个token，这就要用到掩蔽自注意力（masked self-attention），掩蔽自注意力可以通过一个掩码（mask）来阻止每个位置选择其后面的输入信息（在训练阶段非常重要）。

详细的流程或代码参考 MiniMind

注意：

  • 在生成任务中，我们通常只关心下一个词是什么，所以只取序列中最后一个位置的向量来解码。前面的向量虽然包含了丰富信息，但此刻是给未来的词准备的。
  • 在推理（生成）阶段，对于Prompt中的非最后一个Token，我们只需要计算并缓存它们的Key和Value（KV Cache），而无需为它们重复计算完整的注意力。

非自回归解码器

是一种并行生成序列的模型，与自回归（AR）解码器逐步生成的方式不同，它一次性输出整个序列，显著提升了推理速度，但可能牺牲生成质量。

例如：给编码器一堆 <BOS> 的词元。假设输出的句子的长度有上限，绝对不会超过 300 个字。给编码器 300 个 <BOS>，就会输出 300 个字，输出 <EOS> 右边的的输出就当它没有输出。

编码器-解码器注意力

编码器的最终输出（即最后一层的隐藏状态）会作为Key和Value输入到解码器的每一层的交叉注意力（Cross-Attention）模块中。

流程总结

  1. 输入预处理

    • Tokenization：文本 → Token IDs（如 [I, love, NLP] → [1045, 2293, 4263]）。

    • Embedding：Token IDs → 词向量（d_model 维，如 512 维）。

    • Positional Encoding：加入位置信息 → 最终输入向量 X = WordEmbedding + PositionalEncoding。

      位置编码的区别

      • Transformer（原始模型）：

        • 使用 固定的数学函数（正弦和余弦函数）生成位置编码，与输入嵌入（token embeddings）相加。

        • 公式： \[ PE_{(pos, 2i)} = \sin\left(\frac{pos}{10000^{2i/d_{\text{model}}}}\right), \quad PE_{(pos, 2i+1)} = \cos\left(\frac{pos}{10000^{2i/d_{\text{model}}}}\right) \]

          其中 pos 是位置，i 是维度索引，dmodel 是模型维度。

        • 这种编码是确定性的，无需学习，能捕捉绝对位置和相对位置关系。

      • BERT：

        • 使用 可学习的位置嵌入（Learned Positional Embeddings），即每个位置（如最大长度512）对应一个随机初始化的向量，通过训练更新。
        • 位置嵌入与 token embeddings 相加，类似 Transformer，但嵌入值是模型学到的，而非预设的数学函数

  2. Encoder 阶段（N 一般是 6 层）

    • 每层执行：
      • 多头自注意力（计算全局依赖）
      • 前馈网络 FFN（非线性变换）
      • 残差连接 + LayerNorm（稳定训练）
    • 输出：上下文感知的编码表示 Z（供 Decoder 使用）。

  3. Decoder 阶段（N 一般是 6 层）

    • 每层执行：
      • 掩码自注意力（防止未来信息泄漏）
      • 残差连接 + LayerNorm
      • Encoder-Decoder 注意力（关注 Z）
      • 残差连接 + LayerNorm
      • 前馈网络 FFN：先升维再降维（可以理解为 让相关特征更突出，参考视频）
      • 残差连接 + LayerNorm
    • 输出：解码后的隐藏状态 H。

  4. 输出生成

    • 线性层：将 H 投影到词表大小维度（如 50,000 维）。
    • Softmax：生成概率分布 → 选择概率最高的 Token 作为输出（或采样）。

  5. 自回归（Autoregressive）

    • 若为生成任务（如 GPT），将当前输出作为下一步输入，循环直至结束。

BERT和GPT

  • BERT（Bidirectional Encoder Representations from Transformers）——填空
    • 双向编码器：使用Transformer的Encoder部分，能同时利用上下文信息（左右双向上下文）。
    • 适合理解型任务（如文本分类、实体识别、问答）。
    • 结构：Transformer编码器 + MLM(随机掩码) / NSP(下一句预测)
  • GPT（Generative Pre-trained Transformer）——预测下一个词
    • 单向解码器：使用Transformer的Decoder部分（掩码自注意力），没有交叉注意力（因为没有编码器的输出作为输入），仅依赖左侧上下文生成文本。
    • 适合生成型任务（如文本生成、对话、续写）。

Post-Norm 和 Pre-Norm

在 Transformer 及其变体模型中，Pre-Norm 和 Post-Norm 是两种不同的层归一化（Layer Normalization, LayerNorm）放置方式，直接影响模型的训练稳定性、收敛速度和性能。以下是它们的详细对比：

1. Post-Norm（原始 Transformer 设计）

  • 结构顺序：

    残差连接 → LayerNorm

    即：
    \[ \text{Output} = \text{LayerNorm}(x + \text{Sublayer}(x)) \]

    • 例如：多头注意力或前馈网络（FFN）的输出先与输入残差连接，再进行归一化。

  • 特点：

    • 原始 Transformer（论文中的编码器/解码器）和 GLM（由智谱开发的主流中文 LLM 之一）使用 Post-Norm。
    • 训练初期可能不稳定（梯度方差大），需要配合 warm-up 学习率调度。
    • 更深的模型容易因梯度消失/爆炸而难以训练。
    • 理论上有更强的表达能力（因归一化在非线性变换后），但实际需要精细调参。

2. Pre-Norm（主流变体设计）

  • 结构顺序：

    LayerNorm → 残差连接

    即： \[ \text{Output} = x + \text{Sublayer}(\text{LayerNorm}(x)) \]

    • 先对输入归一化，再通过子层（如注意力或 FFN），最后与原始输入残差连接。

  • 特点：

    • BERT、GPT 等现代模型普遍采用 Pre-Norm。
    • 训练更稳定，无需 warm-up 也能快速收敛。
    • 支持更深的网络（如 100+ 层），梯度传播更顺畅。
    • 牺牲少量理论表达能力，换取实际训练的鲁棒性。

3. 核心区别对比

特性 Post-Norm Pre-Norm
归一化位置 残差连接后 残差连接前（子层输入）
训练稳定性 依赖 warm-up，易不稳定 更稳定，适合深层模型
梯度传播 可能梯度消失/爆炸（深模型） 梯度更平缓，易于优化
表达能力 理论上更强（非线性后归一化） 稍弱，但实际差距小
典型应用 原始 Transformer BERT、GPT、T5 等现代模型

4. 其他变体与改进

  • ReZero Norm：将残差连接乘以可学习的标量（初始为 0），逐步增加非线性强度，结合两者优点。
  • Sandwich Norm：在子层前后都加入 LayerNorm（如 Norm → Sublayer → Norm → 残差），进一步提升稳定性。
  • DeepNorm（微软）：在 Post-Norm 基础上调整残差权重，用于极深模型（如 1000 层）。

LLM

三个阶段——Pretrain、SFT 和 RLHF

大语言模型（LLM）的训练通常分为三个关键阶段，每个阶段有不同的目标和方法：

1. Pretraining

预训练

目标： 学习语言的通用知识（语法、语义、世界常识等）

过程：

  • 利用大规模通用语料（如网页、书籍、百科等）进行自监督学习。
  • 训练任务通常是：
    • 因果语言建模（Causal LM）：预测下一个词（GPT 风格）
    • 掩码语言建模（Masked LM）：预测被遮盖的词（BERT 风格）

特点：

  • 模型尚未特定任务对齐，仅有通用语言能力
  • 参数量巨大（几十亿到千亿级别）
  • 训练耗费算力与数据

2. SFT

Supervised Fine-Tuning，监督微调

目标： 将模型调整为能按照人类期望完成具体任务（如问答、摘要）

过程：

  • 使用人工标注的高质量指令-响应对进行监督微调（如 Alpaca、OpenAI InstructGPT 数据）

  • 输入是类似：

特点：

  • 显著提升对“任务”或“指令”的响应能力
  • 强化了与人类沟通的能力，但输出可能仍存在幻觉或不可靠性

注意指令微调和 Prompt 工程的区别

项目 指令微调（Instruction Tuning） Prompt 工程（Prompt Engineering）
是否训练模型 ✅ 需要重新训练或微调模型 ❌ 不训练，直接用
成本 高：需要 GPU、数据、训练资源 低：只需要会写 prompt
是否修改模型参数 ✅ 修改参数 ❌ 不修改
应用阶段 训练阶段（Finetune） 推理阶段（Inference）
优点 长期效果好、可扩展 灵活、快速试验
缺点 昂贵、不易调试 不稳定、易受上下文影响

3. RLHF

Reinforcement Learning with Human Feedback

目标： 利用人类偏好优化模型行为，提升对齐程度（alignment）

过程：

  1. 收集人类偏好数据： 给出同一个输入，模型生成多个回答，由人类标注哪个更好。

  2. 训练奖励模型（RM）： 学习如何从输入和输出中打分。

    • 让 SFT 模型对同一个问题生成多个回答（如 A 和 B）
    • 请人类标注员判断：哪个回答更好？（A > B）
    • 用这些“偏好对”训练一个 Reward Model
    • RM 的输入是“问题 + 回答”，输出是一个分数（如 1~10 分）

  3. 强化学习（PPO 等）： 使用奖励模型对主模型进行强化学习优化。

    • 将 SFT 模型作为策略（Policy）
    • 每次生成一个回答 → RM 给出奖励分数
    • 使用 PPO（Proximal Policy Optimization） 算法更新策略参数
    • 目标：最大化奖励分数
    • 通常加一个 KL 惩罚，防止模型偏离 SFT 模型太远（避免过度优化）

特点：

  • 对齐人类偏好（例如：更礼貌、更安全、更有逻辑）
  • 可以显著改善模型的对话体验和安全性（ChatGPT、Claude 等都采用该方式）
