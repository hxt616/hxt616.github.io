目标和

发表于 2024-03-11 分类于 代码随想录 ， LeetCode
本文字数： 1.2k 阅读时长 ≈ 1 分钟

题目地址：https://leetcode.cn/problems/target-sum/

没思路，直接参考代码随想录。

将原数组分为两部分：正数之和、负数之和。由正数 + 负数 = target 和 负数 = 正数 - sum 这两个式子可推出正数 = (target+sum)/2；接着就是利用0-1背包问题的思想解决问题。

  • 先计算数组元素之和sum，然后判断sum+target是否为偶数，若为奇数则原数组不可能满足条件；若sum<target也说明原数组不满足条件

  • dp[i][j]：前i个元素能够组成的 和为j 的组合个数

  • 递推公式：由于这题求的是组合数，因此需要累加

    则如果不选nums[i-1](即第i个元素)，方案数是dp[i−1][j]，如果选nums[i-1]，方案数是dp[i−1][j−nums[i-1]](即上一个满足条件的方案数)，此时有dp[i][j] = dp[i−1][j] + dp[i−1][j−nums[i-1]]。

  • 初始化：这题的初始化比较特别，dp[0][0]需要置为1，其他元素置为0

    dp[0][0]的意义：当没有任何元素可以选取时，元素和只能是0，对应的方案数是1

  • 遍历顺序：需要从dp[1][0]开始遍历

    下标j从0开始遍历

  • 先定义结构体，对应每个字符串，存放‘0’的个数和‘1’的个数

  • 定义结构体数组，记录每个字符串中‘0’、‘1’的个数

  • dp[i][j][k]：表示前i个字符串中包含j个‘0’和k个‘1’的最大字符串个数

  • 递推公式：当j和k分别大于当前字符串中的‘0’和‘1’的个数，则字符串个数加1

  • 初始化：三维数组所有元素初始化为0

  • 遍历顺序：第一层for循环即字符串下标，从i=1开始遍历，pair[i-1]表示第i个字符串

0-1背包

发表于 2024-03-09 分类于 代码随想录
本文字数： 1.5k 阅读时长 ≈ 1 分钟

题目地址：https://kamacoder.com/problempage.php?pid=1046

0-1背包详解：https://www.hello-algo.com/chapter_dynamic_programming/knapsack_problem/

0-1背包中的“0-1”的意思为当前物品只有两种状态——“选择”或“不选择”。先判断第i个物品是否装入背包，再推断出递推式。

  • 物品i的所占空间大小为weight[i-1]，价值为value[i-1]

  • 物品i是否装入背包：若weight[i-1]大于背包当前剩余空间，则无法放入；若小于等于背包当前剩余空间，则需要判断是放入物品i的背包价值大还是不放入物品i的背包价值大

    这里有个疑惑，显而易见肯定是放入物品的背包价值更大，为什么还要比较？

    但其实这里两个操作的前提是不一样的，即这两个操作对应的背包状态不同，不能直接认为放入物品的背包价值更大。

  • 现在就能确定dp[i][j]的意义了，dp[i][j]表示前i个物品在剩余空间为j的背包中的最大价值

  • 初始化：第一行（dp[0][j]，表示放入0个物品，所以价值为0）和第一列（dp[i][0]，表示背包空间为0，无法放入物品，所以价值为0）需要初始化为0

  • 遍历：从dp[1][1]开始遍历，表示（背包空间为1，第1个物品选择/不选择）背包最大价值

这题的难点在于找递推式，一开始我是看能不能通过节点数对应的二叉搜索树的种数找出规律，但是没发现规律。

想了一会准备从二叉搜索树的性质入手，假设给定节点值1, 2, 3, 4, 5, 6，每次取不同的值作为根节点，则每次划分的左右子树包含的节点数不同。

先选定1作为根节点，根据左右子树计算二叉搜索树的种数，即将左子树种类数乘以右子树种类树；然后以2为根节点计算，以此类推，同时累加 以不同数值为根节点 所计算出的二叉搜索树种数，得到结果。
