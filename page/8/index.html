删除二叉搜索树中的节点

发表于 2024-03-01 分类于 代码随想录 ， LeetCode
本文字数： 1.1k 阅读时长 ≈ 1 分钟

题目地址：https://leetcode.cn/problems/delete-node-in-a-bst/

对于二叉搜索树的删除有下面几种情况：

（1）叶结点：直接删除

（2）只有一棵子树：让子树成为 z 的父结点的子树

（3）有左右两个子树：令右子树最左下结点/直接后继（或左子树最右下结点/直接前驱）代替 z，然后删除直接后继/直接前驱结点

注意：删除一个结点可能会影响多个结点，因此可能要操作多次

由上述情况可知可以使用递归比较方便

二叉树的最近公共祖先

发表于 2024-02-29 分类于 代码随想录 ， LeetCode
本文字数： 927 阅读时长 ≈ 1 分钟

题目地址：https://leetcode.cn/problems/lowest-common-ancestor-of-a-binary-tree/

这题用到了回溯的思想，所以用后序遍历（左-右-中），可以根据左右子树的返回值处理根节点。分为两种情况：

  1. p、q在二叉树的两边

    当左、右子树均不为空，则该节点就是最近的公共祖先。

    例如上图中，p->val = 7，q->val = 0，这两个节点的公共祖先是节点3，模拟后序遍历并不断回溯的过程，发现直到节点3时，左右子树均不为空。（模拟代码的过程）

  2. p、q在二叉树的同一边

    若p是q的祖先，则直接返回p；否则返回q

    若p->val = 5，q->val = 7；后序遍历，先找到左子树中的p节点，则直接返回，此时left = p；然后遍历根节点的右子树，返回值为空，即right=NULL；最后回溯到根节点，则返回不为空的节点，即p。

    这个过程中q节点并没有被访问。

代码思路如下：

  • 若遍历过程中遍历到了空结点、p或q，则返回当前节点
  • 遍历当前节点的左子树
  • 遍历当前节点的右子树
  • 若当前节点的左右子树不为空，则说明当前节点就是最近公共祖先
  • 若一个子树为空另一个不为空，则返回不为空的子树；否则返回NULL

本文字数： 1.3k 阅读时长 ≈ 1 分钟

题目地址：https://leetcode.cn/problems/find-mode-in-binary-search-tree/submissions

思路：

  • 先通过中序遍历得到该二叉树的有序序列
  • 创建一个结构体数组保存不同节点值以及该节点值出现的频率
  • 遍历有序序列填充结构体数组
  • 遍历结构体数组，记录当前频率最大值，若出现更大值则将结果数组清空，填入新的节点值；若相等则将节点值加入结果数组；若小于则continue

发表于 2024-02-29 分类于 代码随想录 ， LeetCode
本文字数： 728 阅读时长 ≈ 1 分钟

题目地址：https://leetcode.cn/problems/validate-binary-search-tree

二叉搜索树（又称二叉排序树）的特点如下：

  • 左子树 < 根结点 < 右子树
  • 中序遍历可得到有序序列
  • 从根结点开始比较，若关键字小于根结点则进入左子树，否则进入右子树
  • 可使用递归，但执行效率低（空间复杂度高）；非递归使用循环
  • 结点的插入：插入的结点一定是叶子结点，可使用递归实现；若存在相同的关键字，插入失败

这题使用中序遍历得到序列，然后判断该序列是否是升序的

错误思路：递归判断每个子树是否满足“左孩子 < 根结点 < 右孩子”，每个子树是满足该条件的，但是下面这种情况就会误判

从中序与后序遍历序列构造二叉树

发表于 2024-02-29 分类于 代码随想录 ， LeetCode
本文字数： 880 阅读时长 ≈ 1 分钟

题目地址：https://leetcode.cn/problems/construct-binary-tree-from-inorder-and-postorder-traversal

唯一地确定一棵二叉树：

  1. 先序和中序：先序确定根结点，中序确定左右子树
  2. 后序和中序：同上
  3. 层序和中序：层序遍历确定根结点

显然这题还是使用递归的思想，中序遍历是“左-中-右”，后序遍历是“左-右-中”，根据左右子树的顺序特点确定二叉树，思路如下：

  • 先写好退出递归的条件，即当序列数组为空时，返回NULL
  • 然后开辟一个节点的空间，将当前的根节点（即后序遍历序列最后一个元素）存放在开辟的空间中
  • 遍历中序遍历序列找到该根节点的位置，从该处分为左右子树，并将当前根节点的左右指针指向两个子树
  • 接着分别对左子树和右子树进行上述操作（注意拆分后的序列的边界）

路径总和

本文字数： 764 阅读时长 ≈ 1 分钟

题目地址：https://leetcode.cn/problems/path-sum/

这题用到了回溯的思想，若使用迭代法，则每次回溯时需要减去刚才保存的节点值，再继续；而递归本身就有回溯的功能

  • 当遍历到叶子节点时，比较路径上的节点值之和与目标值是否相等
  • 若不相等则返回false，回到上一层继续遍历
  • 若相等则返回true，这里定义两个bool类型变量存放左、右路径的布尔值

注意这里不能直接定义bool l = preOrder(root->left, targetSum, path);，同时需要注意对l和r进行初始化。

找树左下角的值

本文字数： 885 阅读时长 ≈ 1 分钟

题目地址：https://leetcode.cn/problems/find-bottom-left-tree-value

这题用迭代法更方便，迭代法思路：每次记录当前层的第一个节点，直到最后一层。

我使用的是递归法，思路如下：

  • 先定义一个全局变量（结构体）用于存放最深的层数和该层最左边节点的值

  • 使用前序遍历，按照“根-左-右”顺序遍历二叉树

  • 按照该顺序遍历，若当前节点为叶节点，则比较深度大小判断是否需要记录当前节点值

    这里不需要判断是左叶子节点还是右叶子节点，因为同一层的叶子节点会先记录左节点，然后再遍历右节点，此时深度值已经更新，所以同一层的右节点不会记录下来

  • 若当前节点有孩子节点，则继续遍历
