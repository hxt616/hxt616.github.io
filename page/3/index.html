先介绍文件共享的知识

文件共享

硬链接

  • 基于索引结点
  • 不同用户的目录项指向同一个索引结点
  • 链接计数count

软链接

  • 创建Link型(快捷方式)文件——记录文件存放路径
  • 共享文件被删除，Link型文件依然存在，但是会查找失败
  • 软链接索引结点中的引用计数值应该恒为1

建立符号链接时，引用计数不是复制原文件而是必为1

解决方案

vscode插件文件夹位置一般为C:\Users\{username}\.vscode\extensions

  • 先该文件夹剪切到其他位置，例如D:\extensions

    或者将文件夹内的文件剪切/复制到目标文件夹下（即D:\extensions），然后将原文件夹(即C:\Users\{username}\.vscode\extensions)删除

  • 然后回车，搞定

mklink用法：MKLINK [[/D] | [/H] | [/J]] Link Target

参数 可选/必须 说明
/D 可选 创建目录符号链接而不是文件符号链接（默认为文件符号链接）
/H 可选 创建硬链接而不是符号链接
/J 可选 创建目录连接点
Link 必选 指定新的符号链接名称
Target 必选 指定新符号链接引用的路径（绝对路径或相对路径均可）

今天是3月25日，科软的拟录取名单公布了，终于考上了😃！！！

个人背景

本科就读于浙江的一所双非，专业是计算机科学与技术。大一到大三成绩还行，不过都是靠期末突击，学的也不怎么深。

高数和线代都是大一学的，早忘光了；数据结构期末考了五十几分，还是靠平时分给我拉上来的；计网、操作系统、计组都是大三上学的，学的很浅，期末考试也是靠考前突击过的。

参赛经历：参加过一些大大小小的CTF竞赛，基本只会签到题和一些简单题😓，大三下抱到大腿拿了个国奖。我大学期间就获得过这一个奖项。

择校

一开始考虑的是合工大和杭电，后来无意中加了科软群，本来是看热闹的，后面了解到科软是无导师制的，培养方案很适合我。三月底，看着群里的拟录取截图，备受鼓舞，所以我就下定决心了​​。

初试分数下来一看383，好像还行。后来复试线一出，376分，倒吸一口凉气，我这分数基本就是擦线了，排名也挺靠后，不过好在复试一切顺利。

还是建议大家理性选择，除了考虑地理位置、学校实力、招收人数、培养方案，还要考虑初试复试占比，如果觉得复试没有什么优势，我觉得还是选择6:4和7:3的学校。

复习建议

经验贴并非现成的复习方案，需要根据自身灵活调整，希望大家理性看待，不要盲目模仿。

我是从3月1日开始正式复习的，如果基础不好，还是建议早点开始。

  • 静态成员和方法由static修饰，属于类，可以通过类名直接访问，无需创建类的实例。

primitive type和reference type

发表于 2024-04-05 分类于 CS61B ， Java
本文字数： 1.1k 阅读时长 ≈ 1 分钟

数据类型可以分为两类：基本数据类型（primitive type）和引用数据类型（reference type）。

基本数据类型是指整数、浮点数、布尔值等（byte、short、int、long、 float、 double、 boolean、char）。这些类型的值是直接存储在内存中的，它们的大小是固定的，因此它们的操作速度非常快。

引用数据类型是指对象、数组、函数等类型。这些类型的值是存储在内存中的地址，而不是直接存储在变量中。因此，当我们操作引用类型时，实际上是在操作存储在内存中的地址，而不是实际的值。这使得引用数据类型的操作速度相对较慢。

一个重要的区别是，基本数据类型的值在传递给函数或赋值给另一个变量时，会被复制一份。而引用数据类型的值在传递给函数或赋值给另一个变量时，只会复制它的地址，而不是它的值。这意味着，如果我们修改了一个引用数据类型的值，它会影响到所有引用该值的变量。
