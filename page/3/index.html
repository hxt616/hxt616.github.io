这题的难点在于，不同顺序的序列属于不同排列，因此不能单纯套用完全背包代码。所以这里的dp[i][j]的意思不是前 i 个数能组成和为 j 的组合数。

  • dp[i][j]：第一个放入的物品为物品 0-i ，恰好装满容量为 j 的背包方式数

  • 递推公式：

    其中dp[numsSize][j-nums[i-1]]表示的是第一个放入的是物品 i ，恰好装满容量为 j 的背包的方式数，[numsSize]表示剩下的物品从 1~numsSize 中选择

  • 初始化：dp[0][0]初始化为1

  • 遍历顺序：由于dp[i][j]取决于

    • dp[i - 1][j] ：故从上往下遍历。
    • dp[numsSize][j - numsSize[i-1]] ：即第j列的值取决于第j - nums[i-1]列最后一行(numsSize)的值，故必须从左往右遍历。且必须先遍历背包再遍历物品才能保证遍历到dp[i][j]时，dp[numsSize][j - nums[i-1]]的值已经被计算过。

完全背包

发表于 2024-03-12 分类于 代码随想录
本文字数： 838 阅读时长 ≈ 1 分钟

题目地址：https://kamacoder.com/problempage.php?pid=1052

详解：https://www.hello-algo.com/chapter_dynamic_programming/unbounded_knapsack_problem/

完全背包问题与0-1背包问题的区别就在于物品是否可以重复选择。完全背包问题中，每种物品的数量是无限的，将物品 i 放入背包后，仍然可以从前 i 个物品中选择。

dp[i][j]表示前 i 个物品能够放入容量为 j 的背包所能达到的最大价值，分为两种情况：

  • 不放入物品 i，此时dp[i][j] = dp[i-1][j]

  • 放入物品 i，此时dp[i][j] = max(dp[i-1][j], dp[i][j-wgt[i-1]]+val[i-1]

    这里第二个表达式中是dp[i]，而0-1背包问题中是dp[i-1]

将原数组分为两部分：正数之和、负数之和。由正数 + 负数 = target 和 负数 = 正数 - sum 这两个式子可推出正数 = (target+sum)/2；接着就是利用0-1背包问题的思想解决问题。

  • 先计算数组元素之和sum，然后判断sum+target是否为偶数，若为奇数则原数组不可能满足条件；若sum<target也说明原数组不满足条件

  • dp[i][j]：前i个元素能够组成的 和为j 的组合个数

  • 递推公式：由于这题求的是组合数，因此需要累加

    则如果不选nums[i-1](即第i个元素)，方案数是dp[i−1][j]，如果选nums[i-1]，方案数是dp[i−1][j−nums[i-1]](即上一个满足条件的方案数)，此时有dp[i][j] = dp[i−1][j] + dp[i−1][j−nums[i-1]]。

  • 初始化：这题的初始化比较特别，dp[0][0]需要置为1，其他元素置为0

    dp[0][0]的意义：当没有任何元素可以选取时，元素和只能是0，对应的方案数是1

  • 遍历顺序：需要从dp[1][0]开始遍历

这题我一开始的思路是先排序，再用二分法分别计算左右两个子数组的元素和，后来发现对于类似于[1, 1, 2, 2]这样的数组无法判断。

代码随想录使用的是0-1背包问题的方法，思路如下：

  • 先计算整个数组元素之和sum
  • 然后判断sum/2是否为偶数，若为奇数则直接返回false，否则将sum/2作为背包最大容量
  • dp[i][j]表示的是前i个元素在剩余空间为j的背包中的最大价值
  • 初始化：第一行和第一列初始化为0
  • 从dp[1][1]开始遍历，代码基本与0-1背包相同
  • 最后判断dp[numsSize][sum/2]所能装的最大值是否等于sum/2，若等于则返回true
