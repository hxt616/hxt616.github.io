最大子数组和

发表于 2024-03-05 分类于 代码随想录 ， LeetCode
本文字数： 676 阅读时长 ≈ 1 分钟

题目地址：https://leetcode.cn/problems/maximum-subarray

题干比较好理解，最简单的做法是暴力求解，使用二重for循环计算出所有的情况，然后记录最大值。这种方法的时间复杂度是O(n2)，超时。

根据代码随想录，使用贪心算法的时间复杂度是O(n)，只需一个for循环即可。这里贪心的思想是：若当前遍历到nums[i]元素，通过该元素前面的子数组之和的值，来判断是继续增加子数组元素还是直接截断。

  • 若子数组之和count大于0，则count+nums[i] > nums[i]，子数组可以增加元素
  • 若子数组之和count小于等于0，则count+nums[i] <= nums[i]，子数组需要截断，从当前元素nums[i]开始计算子数组之和（注意这个过程中count需要重置为0）

非递减子序列

发表于 2024-03-04 分类于 代码随想录 ， LeetCode
本文字数： 1.6k 阅读时长 ≈ 1 分钟

题目地址：https://leetcode.cn/problems/non-decreasing-subsequences

这题与子集Ⅱ很类似，但是子集Ⅱ不要求解集按顺序存储，因此可以先对原数组进行排序再进行操作；而这题要求按照原数组的顺序寻找升序子序列，因此不能对原数组进行排序，否则会影响结果。

因此该题的处理方法与 子集Ⅱ 不同，使用一个辅助数组存放当前层遍历过的元素，通过判断当前元素是否在该层出现过，从而进行去重。

先介绍KMP算法

KMP算法

参考：https://www.zhihu.com/question/21923021

原理

  • 时间复杂度：O(n+m)

  • 主串不回退

  • 前缀：除最后一个字符之外的头部子串

  • 后缀：除第一个字符之外的尾部子串

  • 部分匹配值（PM）：最长相等前后缀长度

  • 遇到不匹配的字符时：移动位数（从当前位置向左移动） = 已匹配的字符数 - 已匹配的部分匹配值(不匹配前一位的对应前缀表中的值)

    模式串指针指向新的位置即pos = 前缀表[pos-1]

    前缀表[pos-1]就是已匹配的部分匹配值

  • next 数组：即当前元素对应的该元素前面的子串的部分匹配值,若当前元素P[2]与主串不匹配，则模式串的指针跳转到P[next[2-1]]（即P[1]）

    注意这篇文章的前提：前缀表就是next数组

    还有其他的表示方法，如下（本文不讨论下面的情况）：

构造next数组：

  • 设模式串为P，假设 next[0], next[1], ... next[x-1] 均已知，求 next[x]

  • 即此时已遍历到模式串 P[x] 元素，需要比较已匹配前缀的后一个字符P[now]与P[x]是否相同：(next[x-1] = now)

    1. 若 P[now] = P[x] ，则 next[x] = next[x-1] + 1

    1. 若 P[now] != P[x] ，则需要缩小 now (即回退已匹配前缀)，now应该改成——使得 A的k-前缀等于B的k-后缀 的最大的k，而B的后缀等于A的后缀（因为A、B是相等的子串），所以只需找到A的k-前缀等于A的k-后缀的最大的k即A的最长公共前后缀的长度，也就是 next[now-1]

    1. 更新 now ：now = next[now-1] ，然后继续比较 P[now] 和 P[x]
