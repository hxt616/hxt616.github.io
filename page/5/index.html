单调递增的数字

发表于 2024-03-07 分类于 代码随想录 ， LeetCode
本文字数： 733 阅读时长 ≈ 1 分钟

题目地址：https://leetcode.cn/problems/monotone-increasing-digits

我的思路是，先将每一位数保存在一个数组中，然后遍历数组进行判断。这题只要想明白变化规律就很简单。

  • 从低位开始遍历，将每一个数字保存到辅助数组number中
  • 然后从原数的低位开始遍历数组（顺序遍历数组），遇到不符合条件的数字，则该数字减1，然后该数字之后的低位数字全置为9
  • 最后再次遍历变化后的数组，累加（不是简单累加，每一次需要乘10）得到结果

划分字母区间

发表于 2024-03-07 分类于 代码随想录 ， LeetCode
本文字数： 751 阅读时长 ≈ 1 分钟

题目地址：https://leetcode.cn/problems/partition-labels/

我原本的思路类似于前面两题用最少数量的箭引爆气球、无重叠区间，先将每个字母对应的起始位置和结束位置记录下来，然后按照起始位置升序，接着比较结束位置。

这个思路是正确的，但是我不会使用qsort对结构体进行排序，所以就按照代码随想录的思路完成了该题。

  • 定义一个辅助数组p[26]（哈希表），将字母的终止位置记录为辅助数组对应下标（a-0,b-1...）的元素
  • 遍历字符串，记录当前最大的终止位置right，当遍历位置与right重合时，说明可以在此处进行划分，记录划分长度right-left+1，同时更新left = right+1
  • 重复第二步操作

用最少数量的箭射爆气球

发表于 2024-03-07 分类于 代码随想录 ， LeetCode
本文字数： 807 阅读时长 ≈ 1 分钟

题目地址：https://leetcode.cn/problems/minimum-number-of-arrows-to-burst-balloons/

通过给定坐标的范围判断可同时射爆几个气球，并记录射出的箭的数量。

  • 先按照坐标的起始值升序排列，若起始值相同则比较终止值
  • 使用变量tmp记录第一个气球坐标的终止值
  • 从下标为1的气球开始for循环遍历，若tmp大于等于当前气球的起始值，则说明两个气球有重合部分，同时更新tmp为终止值较小的那个值
  • 若tmp小于等于当前气球起始值，则需要增加弓箭数量

加油站

发表于 2024-03-06 分类于 代码随想录 ， LeetCode
本文字数： 842 阅读时长 ≈ 1 分钟

题目地址：https://leetcode.cn/problems/gas-station/

直接暴力求解，通过二重循环，遍历以不同加油站为起点的路径，难点在于如何用循环模拟出数组环路（从当前元素向后遍历直到再次遍历到该元素）

  • 第一重for循环以不同加油站为起点，每更换一个起点，sumGas和sumCost都初始化为0，统计以当前加油站为起点的路径上的累加的油量和消耗的油量

  • 第二重循环使用while，i从0开始，(start+i) % gasSize表示路径上的加油站，i的范围是 0 ~ gasSize-1 ，正好能够形成一个环路

  • 若当前路径上累加的油量小于消耗的油量，则退出当前while循环，以下一个加油站为起点继续；否则，当前路径成立并判断下一个加油站是否满足条件（i++）

  • 退出while循环后，判断当前路径是否包含所有加油站，若包含，则返回start起点加油站；否则以第一个无法到达的加油站为起点继续（for循环）

K次取反后最大化的数组和

发表于 2024-03-06 分类于 代码随想录 ， LeetCode
本文字数： 676 阅读时长 ≈ 1 分钟

题目地址：https://leetcode.cn/problems/maximize-sum-of-array-after-k-negations/

注意题中的取反可以作用于同一个元素。首先需要对原数组进行升序排序，一共分为三种情况：

  • 最小值为负数：若k>0，则将最小值取反
  • 最小值为0：取反作用在0上，相当于不变，因此可直接计算数组和
  • 最小值为正数：若k>0，则将最小值取反

上述步骤对应局部最优，使用for循环每次取反后对数组重新排序，最后得到全局最优

贪心算法：

  • 上述变量中，maxIndex需要不断比较以保持最大值，而当遍历到的元素下标为currentMaxIn时（currentMaxIn未到达数组最后一个下标且还没更新maxIndex），maxIndex还未到达数组最大下标，则需要增加步数

题目地址：https://leetcode.cn/problems/jump-game

这题我又是理解错意思了，我以为跳跃的长度就是数组对应的元素，但其实数组对应的元素表示的跳跃的最大值，也就是说跳跃长度范围是0~nums[i]。

贪心算法：判断跳跃长度能否覆盖整个数组，这里的难点在于跳跃长度为0的元素

每次记录能够到达的最远的数组下标，当遍历到该元素时，判断跳跃距离是否为0

  • 若为0，则无法到达最后一个下标
  • 若不为0，则继续

买卖股票的最佳时机Ⅱ

发表于 2024-03-05 分类于 代码随想录 ， LeetCode
本文字数： 760 阅读时长 ≈ 1 分钟

题目地址：https://leetcode.cn/problems/best-time-to-buy-and-sell-stock-ii

这题还是用贪心算法比较方便。(类似于摆动序列，通过峰值判断是否进行操作)

  1. 假设当前买入的股票价格是buy
  2. 遍历prices剩余的价格（假设此时遍历到prices[i]）
    • 若prices[i]大于等于buy，则更新出售价格sell，重复上述步骤直到第一个峰值（假设为prices[k]），则进行出售
    • 若prices[i]小于buy，则以buy的价格出售（至少保证不亏钱）
  3. 出售完成后需要更新买入价格和出售价格为prices[k+1]，然后重复上述步骤

代码随想录提供的思路更加简洁，计算每一天的利润（除了第一天），然后把结果为正数的利润加起来就是最大利润。
