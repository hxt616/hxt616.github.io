加油站

发表于 2024-03-06 分类于 代码随想录 ， LeetCode
本文字数： 842 阅读时长 ≈ 1 分钟

题目地址：https://leetcode.cn/problems/gas-station/

直接暴力求解，通过二重循环，遍历以不同加油站为起点的路径，难点在于如何用循环模拟出数组环路（从当前元素向后遍历直到再次遍历到该元素）

  • 第一重for循环以不同加油站为起点，每更换一个起点，sumGas和sumCost都初始化为0，统计以当前加油站为起点的路径上的累加的油量和消耗的油量

  • 第二重循环使用while，i从0开始，(start+i) % gasSize表示路径上的加油站，i的范围是 0 ~ gasSize-1 ，正好能够形成一个环路

  • 若当前路径上累加的油量小于消耗的油量，则退出当前while循环，以下一个加油站为起点继续；否则，当前路径成立并判断下一个加油站是否满足条件（i++）

  • 退出while循环后，判断当前路径是否包含所有加油站，若包含，则返回start起点加油站；否则以第一个无法到达的加油站为起点继续（for循环）

K次取反后最大化的数组和

发表于 2024-03-06 分类于 代码随想录 ， LeetCode
本文字数： 676 阅读时长 ≈ 1 分钟

题目地址：https://leetcode.cn/problems/maximize-sum-of-array-after-k-negations/

注意题中的取反可以作用于同一个元素。首先需要对原数组进行升序排序，一共分为三种情况：

  • 最小值为负数：若k>0，则将最小值取反
  • 最小值为0：取反作用在0上，相当于不变，因此可直接计算数组和
  • 最小值为正数：若k>0，则将最小值取反

上述步骤对应局部最优，使用for循环每次取反后对数组重新排序，最后得到全局最优

买卖股票的最佳时机Ⅱ

发表于 2024-03-05 分类于 代码随想录 ， LeetCode
本文字数： 760 阅读时长 ≈ 1 分钟

题目地址：https://leetcode.cn/problems/best-time-to-buy-and-sell-stock-ii

这题还是用贪心算法比较方便。(类似于摆动序列，通过峰值判断是否进行操作)

  1. 假设当前买入的股票价格是buy
  2. 遍历prices剩余的价格（假设此时遍历到prices[i]）
    • 若prices[i]大于等于buy，则更新出售价格sell，重复上述步骤直到第一个峰值（假设为prices[k]），则进行出售
    • 若prices[i]小于buy，则以buy的价格出售（至少保证不亏钱）
  3. 出售完成后需要更新买入价格和出售价格为prices[k+1]，然后重复上述步骤

代码随想录提供的思路更加简洁，计算每一天的利润（除了第一天），然后把结果为正数的利润加起来就是最大利润。

最大子数组和

发表于 2024-03-05 分类于 代码随想录 ， LeetCode
本文字数： 676 阅读时长 ≈ 1 分钟

题目地址：https://leetcode.cn/problems/maximum-subarray

题干比较好理解，最简单的做法是暴力求解，使用二重for循环计算出所有的情况，然后记录最大值。这种方法的时间复杂度是O(n2)，超时。

根据代码随想录，使用贪心算法的时间复杂度是O(n)，只需一个for循环即可。这里贪心的思想是：若当前遍历到nums[i]元素，通过该元素前面的子数组之和的值，来判断是继续增加子数组元素还是直接截断。

  • 若子数组之和count大于0，则count+nums[i] > nums[i]，子数组可以增加元素
  • 若子数组之和count小于等于0，则count+nums[i] <= nums[i]，子数组需要截断，从当前元素nums[i]开始计算子数组之和（注意这个过程中count需要重置为0）
