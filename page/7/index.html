使用数组指针和指针数组访问二维数组元素

  1. 数组指针

现在来看 (*p)[1] 、 p[1] 和*p[1]的区别：

  • (*p)[1]（是元素）意味着先解引用 p，得到一个数组，然后再取该数组的第二个元素。因为 p 指向 array 的第一个元素，所以 (*p)[1] 等价于 array[0][1]，它访问了数组 array 的第一行的第二个元素。
  • p[1]（是数组）意味着先访问 p 的下一个元素，然后再解引用。由于 p 指向一个包含5个整数的数组，因此 p[1] 将访问 p 后面的第二个数组（假设 array 至少有两行），然后对该数组解引用。这与 array[0][1] 不同，因为它跳过了第一行。
  • *p[1] 在 C 语言中的解释取决于 p 的声明。在你的例子中，p 被声明为指向包含5个整数的数组的指针。因此，*p[1] 表示对 p 的第二个元素（假设 p 指向的数组至少有两个元素）进行解引用，然后再取其第一个元素。*p[1] 等价于 p[1][0]
  1. 指针数组

二叉树的最近公共祖先

发表于 2024-02-29 分类于 代码随想录 ， LeetCode
本文字数： 927 阅读时长 ≈ 1 分钟

题目地址：https://leetcode.cn/problems/lowest-common-ancestor-of-a-binary-tree/

这题用到了回溯的思想，所以用后序遍历（左-右-中），可以根据左右子树的返回值处理根节点。分为两种情况：

  1. p、q在二叉树的两边

    当左、右子树均不为空，则该节点就是最近的公共祖先。

    例如上图中，p->val = 7，q->val = 0，这两个节点的公共祖先是节点3，模拟后序遍历并不断回溯的过程，发现直到节点3时，左右子树均不为空。（模拟代码的过程）

  2. p、q在二叉树的同一边

    若p是q的祖先，则直接返回p；否则返回q

    若p->val = 5，q->val = 7；后序遍历，先找到左子树中的p节点，则直接返回，此时left = p；然后遍历根节点的右子树，返回值为空，即right=NULL；最后回溯到根节点，则返回不为空的节点，即p。

    这个过程中q节点并没有被访问。

代码思路如下：

  • 若遍历过程中遍历到了空结点、p或q，则返回当前节点
  • 遍历当前节点的左子树
  • 遍历当前节点的右子树
  • 若当前节点的左右子树不为空，则说明当前节点就是最近公共祖先
  • 若一个子树为空另一个不为空，则返回不为空的子树；否则返回NULL
