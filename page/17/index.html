GCC

GCC全称 the GNU Compiler Collection ，是一套编程语言编译器，以GPL及LGPL许可证所发行的自由软件，也是GNU计划的关键部分，也是GNU工具链的主要组成部分之一。可处理Fortran、Pascal、Objective-C、Java、Ada，Go与其他语言。

GCC中包含gcc和g++，分别处理C语言和C++。关于这两者的区别可参考http://c.biancheng.net/view/7936.html

gcc使用技巧

  1. 当文件没有扩展名时，可使用 -x 选项指明当前文件的类型。比如当前有个文件demo，当直接使用gcc demo会报错，因为GCC编译器无法识别demo这个没有扩展名的文件，因此使用 gcc -xc demo （将文件当作C语言处理）。

  2. gcc + 文件名，生成对应的可执行文件，默认为a.out 。

  3. -o选项指定生成的可执行文件的文件名。

  4. -E选项可控制GCC编译器仅对源码做预处理操作。
    所谓预处理操作，主要是处理那些源文件和头文件中以 # 开头的命令（比如 #include、#define、#ifdef 等），并删除程序中所有的注释 // 和 /* ... */。注意展开头文件时，仅添加声明，不包含函数的具体实现

    一般将预处理的内容保存在.i后缀的文件中 :

  5. -S选项可令GCC编译器仅将指定文件加工至编译阶段，并生成对应的汇编代码文件。

    以上这两行命令均能生成与原文件名相同的后缀为.s的文件（内容为汇编代码）。

  6. -c选项可让GCC编译器将指定文件加工至汇编阶段（但并不执行链接操作），并生成对应的目标文件。

    -c选项同样对.c和.i文件有用。

  7. gcc执行链接操作。

  8. -g选项可产生调试信息，可以使用gdb调试可执行文件。

  9. -m32选项指定生成32位机器的汇编代码，同理-m64则生成64位机器汇编代码。

SI 和 DI

si 和 di 是8086CPU中和 bx 功能相近的寄存器，si 和 di 不能分成两个8位寄存器来使用。

[bx+si] 和 [bx+di]

[bx+si] 表示一个内存单元，它的偏移地址为 (bx) + (si)（即 bx 中的数值加上 si 中的数值）。

指令mov ax,[bx+si]的数学化描述为：(ax)=((ds)*16+(bx)+(si))。

[bx+si+idata] 和 [bx+di+idata]

[bx+si+idata]表示一个内存单元，它的偏移地址为 (bx) + (si) +idata（即 bx 中的数据加上 si 中的数据再加上 idata）。

指令mov ax,[bx+si+idata]的数学化描述为：(ax)=((ds)*16+(bx)+(si)+idata)。

Base64是一种基于64个可打印字符来表示二进制数据的表示方法。由于2的6次方等于64，正好对应了64个字符，所以每6个比特为一个单元，对应某个可打印字符。

转换的时候，将3 字节的数据，先后放入一个 24 位的缓冲区中，先来的字节占高位。数据不足 3 字节的话，于缓冲器中剩下的比特用 0 补足。

汇编语言（四）——[BX]和loop指令

[bx]

上述指令的功能：bx中存放的数据作为一个偏移地址 EA ，段地址 SA 默认在 ds 中，将 SA:EA 处的数据送入 ax 中，即：(ax)=((ds)*16+(bx))。

该指令的功能则是：将 ax 中的数据送入内存 SA:EA 处，即：((ds)*16+(bx))=(ax)。

我们可以不写新的指令，仅通过改变 bx 中的数值，改变指令访问的内存单元。

Loop指令

CPU执行loop指令时要进行两步操作：

  • (cx)=(cx)-1。
  • 判断 cx 中的值，不为0则转至标号处执行程序，如果为0则向下执行。

由上述可知 cx 中存放着循环次数。

段前缀

可以在访问内存单元的指令中显式地给出内存单元的段地址所在的段寄存器。这些用于访问内存单元的指令中，用于显式地指明内存单元的段地址的 ds:（数据段寄存器）、cs:（代码段寄存器）、ss:（堆栈段寄存器）、es:（附加段寄存器），在汇编语言中称为段前缀。

一段安全的空间

一般的PC机中，DOS方式下，DOS和其他合法的程序一般不会使用0:200 ~ 0:2ff（00200h ~ 002ffh）的256个字节的空间。因此当我们需要直接向一段内存中写入内容时，就使用0:200 ~ 0:2ff这段空间。
