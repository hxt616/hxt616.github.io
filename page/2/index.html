二叉树中的最大路径和

发表于 2024-03-18 分类于 LeetCode
本文字数： 633 阅读时长 ≈ 1 分钟

题目地址：https://leetcode.cn/problems/binary-tree-maximum-path-sum/

这题看着不难，实现起来还是比较复杂，递归关系比较难想到。

  • 先确定终止条件，也就是当前节点为空，直接返回0
  • 然后记录左节点、右节点的最大路径和
  • 接着记录当前路径和（当前节点值+左节点路径和+右节点路径和）
  • 若该路径和大于之前的路径和maxS，则更新maxS
  • 最后需要返回，上一级节点以当前节点为子树，则子树最大路径和为root->val + fmax(leftM, rightM)

组合总和IV

发表于 2024-03-12 分类于 代码随想录 ， LeetCode
本文字数： 914 阅读时长 ≈ 1 分钟

题目地址：https://leetcode.cn/problems/combination-sum-iv/

这题的难点在于，不同顺序的序列属于不同排列，因此不能单纯套用完全背包代码。所以这里的dp[i][j]的意思不是前 i 个数能组成和为 j 的组合数。

  • dp[i][j]：第一个放入的物品为物品 0-i ，恰好装满容量为 j 的背包方式数

  • 递推公式：

    其中dp[numsSize][j-nums[i-1]]表示的是第一个放入的是物品 i ，恰好装满容量为 j 的背包的方式数，[numsSize]表示剩下的物品从 1~numsSize 中选择

  • 初始化：dp[0][0]初始化为1

  • 遍历顺序：由于dp[i][j]取决于

    • dp[i - 1][j] ：故从上往下遍历。
    • dp[numsSize][j - numsSize[i-1]] ：即第j列的值取决于第j - nums[i-1]列最后一行(numsSize)的值，故必须从左往右遍历。且必须先遍历背包再遍历物品才能保证遍历到dp[i][j]时，dp[numsSize][j - nums[i-1]]的值已经被计算过。

完全背包

发表于 2024-03-12 分类于 代码随想录
本文字数： 838 阅读时长 ≈ 1 分钟

题目地址：https://kamacoder.com/problempage.php?pid=1052

详解：https://www.hello-algo.com/chapter_dynamic_programming/unbounded_knapsack_problem/

完全背包问题与0-1背包问题的区别就在于物品是否可以重复选择。完全背包问题中，每种物品的数量是无限的，将物品 i 放入背包后，仍然可以从前 i 个物品中选择。

dp[i][j]表示前 i 个物品能够放入容量为 j 的背包所能达到的最大价值，分为两种情况：

  • 不放入物品 i，此时dp[i][j] = dp[i-1][j]

  • 放入物品 i，此时dp[i][j] = max(dp[i-1][j], dp[i][j-wgt[i-1]]+val[i-1]

    这里第二个表达式中是dp[i]，而0-1背包问题中是dp[i-1]
