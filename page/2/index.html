本文字数： 1.5k 阅读时长 ≈ 1 分钟

题目地址：https://leetcode.cn/problems/binary-tree-level-order-traversal/

层序遍历需要借助队列实现，首先入队根结点，然后出队同时将其左子树和右子树入队，接着出队，同时将当前出队结点的左右子树入队，继续出队，重复之前的操作。

这题的难点在于需要统计每一层的节点数，这里用队尾与队头之差表示。

  • 先定义必要的变量
  • 若二叉树不为空，则将根节点入队
  • while循环判断队列是否为空，嵌套以下内容：
    • 计算当前队列中的节点个数
    • 记录当前队头位置，然后队头节点出队
    • 使用for循环将该层节点值加入结果数组res中，注意循环的边界判断（当前层的节点），然后将下一层的节点入队
    • for循环结束后注意增加层数*returnSize
  • 退出while循环后，为*returnColumnSizes数组分配空间，然后根据记录的不同层的节点个数，为该数组赋值
  • 返回结果数组res

前k个高频元素

发表于 2024-02-17 分类于 代码随想录 ， LeetCode
本文字数： 1.4k 阅读时长 ≈ 1 分钟

题目地址：https://leetcode.cn/problems/top-k-frequent-elements/

代码随想录中的思路是使用优先队列构建小顶堆，每次弹出最小频率的元素，剩余的k个元素就是前k个高频元素，用C++实现比较方便，但是使用C语言就比较繁琐。

使用C语言实现则利用快速排序实现，思路如下：

  • 先定义一个结构体包括元素值和元素频率

  • 声明cmp和cmps函数，作为qsort的参数，实现升/降序排序（通过a、b的顺序判断）

  • 实现主体函数，先使用qsort对原数组进行升序排序，使得相同值的元素相邻。

  • 为结果数组res分配空间，给*returnSize赋值

  • 定义结构体数组，使用for循环统计每个元素出现的个数

  • 使用qsort对结构体的频率部分进行排序，最后将前k个结构体对应的元素值存入res中

滑动窗口最大值

发表于 2024-02-16 分类于 代码随想录 ， LeetCode
本文字数： 1.3k 阅读时长 ≈ 1 分钟

题目地址：https://leetcode.cn/problems/sliding-window-maximum/

这题使用双循环暴力求解会超时，然后想了很久也没弄出来，看了题解也理解了半天。

这题的一个重要知识点就是单调队列，即单调递减或单调递增的队列。

思路如下：

  • 首先分配好结果数组res的空间大小numsSize-k+1，然后定义一个数组queue作为双端队列，该双端队列不是直接存放nums数组中的元素，而是存放对应元素的下标
  • 通过循环先将前k个元素加入queue中，同时要保持该队列为单调递减队列，即入队元素大于队尾元素时，则从弹出队尾元素，直到入队元素小于队尾元素或队列为空，再将当前元素入队。（注意这里入队的是对应元素的下标，上述的描述是为了方便）
  • 将第一个滑动窗口最大元素加入res中
  • 通过循环将剩余元素入队，原理同第二步，入队后需要判断队中元素下标的范围是否大于滑动窗口大小。若大于，则弹出队头元素，队头下标增加，再将当前队头元素加入res中；否则直接将队头元素加入res中。（注意这里入队的是对应元素的下标）

[]比*的优先级更高，p是数组，该数组包含5个指针

二维数组

使用数组指针和指针数组访问二维数组元素

  1. 数组指针

现在来看 (*p)[1] 、 p[1] 和*p[1]的区别：

  • (*p)[1]（是元素）意味着先解引用 p，得到一个数组，然后再取该数组的第二个元素。因为 p 指向 array 的第一个元素，所以 (*p)[1] 等价于 array[0][1]，它访问了数组 array 的第一行的第二个元素。
  • p[1]（是数组）意味着先访问 p 的下一个元素，然后再解引用。由于 p 指向一个包含5个整数的数组，因此 p[1] 将访问 p 后面的第二个数组（假设 array 至少有两行），然后对该数组解引用。这与 array[0][1] 不同，因为它跳过了第一行。
  • *p[1] 在 C 语言中的解释取决于 p 的声明。在你的例子中，p 被声明为指向包含5个整数的数组的指针。因此，*p[1] 表示对 p 的第二个元素（假设 p 指向的数组至少有两个元素）进行解引用，然后再取其第一个元素。*p[1] 等价于 p[1][0]
  1. 指针数组

通过一个循环将二维数组每行的起始地址赋给了指针数组 p 的相应元素

有效的括号

发表于 2024-02-09 分类于 代码随想录 ， LeetCode
本文字数： 1.9k 阅读时长 ≈ 2 分钟

题目地址：https://leetcode.cn/problems/valid-parentheses/

有关括号匹配的题一般都是用栈来实现。

下面是我自己的代码，比较繁琐：

  • 遍历 s 字符串，遇到左括号就入栈，遇到右括号则判断栈内是否为空，若不为空则判断栈顶元素是否为对应的右括号；若不满足条件则返回 false ，满足条件则栈顶元素出栈
  • 循环结束后判断栈内是否为空，若不为空（有多余的左括号）则返回 false

下面介绍代码随想录的解法：

遍历 s 字符串，遇到左括号，则入栈；遇到右括号则与栈顶元素进行配对

  • 第一种情况：已经遍历完了字符串，但是栈不为空，说明有相应的左括号没有右括号来匹配，所以return false

  • 第二种情况：遍历字符串匹配的过程中，发现栈里没有要匹配的字符。所以return false

  • 第三种情况：遍历字符串匹配的过程中，栈已经为空了，没有匹配的字符了，说明右括号没有找到对应的左括号return false
