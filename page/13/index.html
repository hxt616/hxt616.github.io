这题也是408的原题。

解决思路：

  • 分别定义两个指针指向两个字符串首地址
  • 计算出两个字符串的长度
  • 移动长度较大的字符串的指针，使得两个指针指向的节点到尾节点之间的节点个数相同（即链表末尾对齐）
  • 同时移动两个指针，若存在指针相同的节点则返回该节点，否则返回 NULL

长度最小的子数组

发表于 2024-01-02 分类于 代码随想录 ， LeetCode
本文字数： 744 阅读时长 ≈ 1 分钟

题目地址：https://leetcode.cn/problems/minimum-size-subarray-sum/

滑动窗口算是双指针的一种，可以类比计算机网络里的滑动窗口。

主要的思想是：从A[i]开始，向后遍历找到满足条件的最小长度；若能找到，分析可知以A[i]开始的窗口大小只有这一个最小值，然后寻找以A[i+1]开始的最小长度。

  • 初始时，左、右指针指向数组下标为0的元素，窗口大小为最大正整数
  • 右指针右移，比较当前窗口中元素之和与目标值的大小：
    1. 若大于等于target，且当前窗口小于之前的值，则更新窗口大小，同时左指针右移，直到窗口里元素之和小于target
    2. 否则，右指针继续后移
  • 最后比较窗口大小是否等于最大正整数，若等于，则说明不存在符合条件的子数组，返回0；否则返回窗口大小

这里虽然使用了while，但时间复杂度还是O(n)，不是O(n2)
